title: "Obtenir le statut de la vidéo"
openapi: "GET /v1/tasks/{id}"
description: "Récupère le statut et le résultat d'une tâche de génération de vidéo"
---

## Paramètres de chemin

<ParamField path="id" type="string" required>
  L'ID de tâche renvoyé par la requête de création de vidéo. Considérez `id` et `task_id` comme la même identité asynchrone.
</ParamField>

<Info>
Si une réponse de création renvoie `poll_url`, appelez cette URL exacte. Lorsqu'elle pointe vers `/v1/tasks/{id}`, considérez-la comme le point de terminaison canonique et fixe pour le statut.

Si la tâche n'existe plus ou ne peut plus être résolue via le contrat public async-task, AI Sonar renvoie `async_task_not_found` avec le message `Task not found or no longer available.`
</Info>

## Réponse

<ResponseField name="id" type="string">
  Identifiant canonique de la tâche asynchrone.
</ResponseField>

<ResponseField name="task_id" type="string">
  Alias de l'identifiant de tâche asynchrone.
</ResponseField>

<ResponseField name="poll_url" type="string">
  URL de polling préférée lorsque la réponse de création en fournit une.
</ResponseField>

<ResponseField name="billing_transaction_id" type="string">
  ID de transaction de facturation AI Sonar lorsque le règlement est déjà terminé. Il s'agit de l'identifiant utilisé pour le dashboard / le rapprochement, distinct de l'`id` / `task_id` asynchrone.
</ResponseField>

<ResponseField name="status" type="string">
  Statut de la tâche : `pending`, `processing`, `completed`, `failed`.
</ResponseField>

<ResponseField name="progress" type="number">
  Pourcentage d'avancement (0-100).
</ResponseField>

<ResponseField name="video_url" type="string">
  URL de la vidéo générée (lorsqu'elle est terminée).
</ResponseField>

<ResponseField name="video" type="object">
  Charge utile d'une seule vidéo avec `url`, `duration`, `width` et `height` lorsque disponible.
</ResponseField>

<ResponseField name="videos" type="array">
  Plusieurs charges utiles vidéo lorsque la source en amont renvoie plus d'une sortie.
</ResponseField>

<ResponseField name="error" type="string">
  Message d'erreur (en cas d'échec).
</ResponseField>

<ResponseField name="created" type="integer">
  Horodatage de création.
</ResponseField>

<ResponseField name="updated" type="integer">
  Horodatage de la dernière mise à jour.
</ResponseField>

<ResponseField name="model" type="string">
  Modèle utilisé pour la tâche.
</ResponseField>

```python Python
import requests
import time

task_id = "ldtask_aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa"

while True:
    response = requests.get(
        f"https://api.aisonar.dev/v1/tasks/{task_id}",
        headers={"Authorization": "Bearer sk-your-api-key"}
    )

    data = response.json()
    print(f"Status: {data['status']}, Progress: {data.get('progress', 0)}%")

    if data["status"] == "completed":
        print(f"Video URL: {data['video_url']}")
        break
    elif data["status"] == "failed":
        print(f"Error: {data['error']}")
        break

    time.sleep(5)  # Poll every 5 seconds
```

## Bonnes pratiques de polling

- Effectuer un polling toutes les 5 à 10 secondes
- Implémenter un backoff exponentiel pour les tâches longues
- Définir un délai d'attente maximal (par exemple, 10 minutes)
- Gérer le statut `failed` de manière appropriée

```python
import time

def wait_for_video(task_id, max_wait=600, interval=5):
    """Wait for video with timeout."""
    start = time.time()

    while time.time() - start < max_wait:
        response = requests.get(
            f"https://api.aisonar.dev/v1/tasks/{task_id}",
            headers={"Authorization": "Bearer sk-your-api-key"}
        )
        data = response.json()

        if data["status"] == "completed":
            return data["video_url"]
        elif data["status"] == "failed":
            raise Exception(data.get("error", "Video generation failed"))

        time.sleep(interval)

    raise TimeoutError("Video generation timed out")
```
