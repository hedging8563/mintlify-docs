---
title: "Obtenir un modèle"
openapi: "GET /v1/models/{model}"
description: "Récupère une instance de modèle"
---

## Paramètres du chemin

<ParamField path="model" type="string" required>
  L’ID du modèle à récupérer (par ex., `gpt-5.4`, `claude-sonnet-4-6`).
</ParamField>

## Réponse

<ResponseField name="id" type="string">
  Identifiant du modèle.
</ResponseField>

<ResponseField name="object" type="string">
  Toujours `model`.
</ResponseField>

<ResponseField name="created" type="integer">
  Horodatage de création.
</ResponseField>

<ResponseField name="owned_by" type="string">
  Fournisseur du modèle.
</ResponseField>

<ResponseField name="aisonar" type="object">
  Métadonnées publiques spécifiques à AI Sonar. La route de détail peut inclure `category`, `pricing_unit`, `capability_flags`, `supported_operations`, `pricing_provenance`, `public_contract_summary`, le `public_contract` complet et les `agent_preferences` au niveau de la scène pour les instantanés de recommandation non-chat.
</ResponseField>

<RequestExample>
```bash cURL
curl "https://api.aisonar.dev/v1/models/gpt-5.4" \
  -H "Authorization: Bearer sk-your-api-key"
```

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.aisonar.dev/v1"
)

model = client.models.retrieve("gpt-5.4")
print(f"Model: {model.id}, Provider: {model.owned_by}")
```

```javascript JavaScript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-your-api-key',
  baseURL: 'https://api.aisonar.dev/v1'
});

const model = await client.models.retrieve('gpt-5.4');
console.log(`Model: ${model.id}, Provider: ${model.owned_by}`);
```

```go Go
package main

import (
    "encoding/json"
    "fmt"
    "net/http"
)

func main() {
    req, _ := http.NewRequest("GET", "https://api.aisonar.dev/v1/models/gpt-5.4", nil)
    req.Header.Set("Authorization", "Bearer sk-your-api-key")

    resp, _ := http.DefaultClient.Do(req)
    defer resp.Body.Close()

    var model map[string]interface{}
    json.NewDecoder(resp.Body).Decode(&model)
    fmt.Printf("Model: %s, Provider: %s\n", model["id"], model["owned_by"])
}
```

```php PHP
<?php
$ch = curl_init('https://api.aisonar.dev/v1/models/gpt-5.4');

curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => [
        'Authorization: Bearer sk-your-api-key'
    ]
]);

$response = curl_exec($ch);
curl_close($ch);

$model = json_decode($response, true);
echo "Model: {$model['id']}, Provider: {$model['owned_by']}\n";
```

## Gestion des erreurs

Si le modèle n’existe pas, vous recevrez une erreur 404 :

```json
{
  "error": {
    "message": "Model 'invalid-model' not found",
    "type": "invalid_request_error",
    "code": "model_not_found"
  }
}
```

## Métadonnées de recommandation non-chat

Pour les modèles non-chat, la route de détail renvoie aussi `aisonar.agent_preferences.<scene>`.

Utilisez-la pour inspecter :

- `preferred_rank`
- `success_rate_24h`
- `sample_count_24h`
- `status`
- `updated_at`

La réponse publique garde volontairement `basis.channel_id` et `basis.physical_model` à `null` afin que les agents puissent consommer le signal de recommandation sans apprendre les identifiants de routage internes.

## Métadonnées publiques de détail

Par rapport à `GET /v1/models`, la route de détail peut également renvoyer :

- `aisonar.capability_flags`
- `aisonar.supported_operations`
- `aisonar.pricing_provenance`
- `aisonar.public_contract_summary`
- `aisonar.public_contract`
