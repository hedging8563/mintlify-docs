---
title: "Créer une variation d'image"
openapi: "POST /v1/images/variations"
description: "Crée une variation d'une image donnée"
---

## Aperçu

Cet endpoint est conservé pour les clients legacy de variations compatibles OpenAI. Ce n’est pas le chemin recommandé pour les workflows image-to-image actuels ; utilisez `/v1/images/edits` avec `gpt-image-2` ou `/v1/images/generations` avec un modèle image-to-image comme `nano-banana-pro` lorsque possible. Requiert `multipart/form-data`.

## Corps de la requête

**Timeout des requêtes synchrones :** certains fournisseurs d’images routés renvoient l’image finale inline et attendent la fin de la génération. Les requêtes haute résolution ou haute qualité peuvent prendre près d’une minute, voire plus ; configurez donc le timeout de votre client HTTP à au moins `120s`. Si la réponse de création contient `status: "pending"`, `task_id` ou `poll_url`, suivez plutôt le `poll_url` renvoyé.

<ParamField body="image" type="file" required>
  L'image à utiliser comme base pour la variation. Doit être un fichier PNG valide, inférieur à 50 Mo et carré.
</ParamField>

<ParamField body="model" type="string" required>
  Identifiant de modèle obligatoire pour les routes legacy compatibles avec les variations. Il n’y a pas de valeur par défaut implicite ; pour les workflows image-to-image actuels, préférez `/v1/images/edits` ou `/v1/images/generations` avec un modèle actuel.
</ParamField>

<ParamField body="n" type="integer" default="1">
  Le nombre d'images à générer. Doit être compris entre 1 et 10.
</ParamField>

<ParamField body="size" type="string">
  Taille des images générées pour les routes legacy compatibles avec les variations. Les valeurs prises en charge dépendent du modèle sélectionné.
</ParamField>

<ParamField body="response_format" type="string" default="url">
  Le format de retour des images générées. Doit être `url` ou `b64_json`.
</ParamField>

<ParamField body="user" type="string">
  Identifiant unique représentant votre utilisateur final pour la surveillance des abus.
</ParamField>

## Réponse

<ResponseField name="created" type="integer">
  Horodatage Unix de la création des images.
</ResponseField>

<ResponseField name="data" type="array">
  Tableau des variations d'images générées.

  Chaque objet contient :
  - `url` (string) : URL de l'image de variation (si response_format est `url`)
  - `b64_json` (string) : Image encodée en Base64 (si response_format est `b64_json`)
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST "https://api.aisonar.dev/v1/images/variations" \
  -H "Authorization: Bearer sk-your-api-key" \
  -F "model=your-legacy-variation-model" \
  -F "image=@cat.png" \

  -F "size=1024x1024"
```

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.aisonar.dev/v1"
)

response = client.images.create_variation(
    model="your-legacy-variation-model",
    image=open("cat.png", "rb"),
    n=2,
    size="1024x1024"
)

for image in response.data:
    print(image.url)
```

```go Go
package main

import (
    "bytes"
    "fmt"
    "io"
    "mime/multipart"
    "net/http"
    "os"
)

func main() {
    body := &bytes.Buffer{}
    writer := multipart.NewWriter(body)

    writer.WriteField("model", "your-legacy-variation-model")

    file, _ := os.Open("cat.png")
    defer file.Close()
    part, _ := writer.CreateFormFile("image", "cat.png")
    io.Copy(part, file)

    writer.WriteField("n", "2")
    writer.WriteField("size", "1024x1024")
    writer.Close()

    req, _ := http.NewRequest("POST", "https://api.aisonar.dev/v1/images/variations", body)
    req.Header.Set("Authorization", "Bearer sk-your-api-key")
    req.Header.Set("Content-Type", writer.FormDataContentType())

    client := &http.Client{}
    resp, _ := client.Do(req)
    defer resp.Body.Close()

    result, _ := io.ReadAll(resp.Body)
    fmt.Println(string(result))
}
```

## Notes

<Note>
  Il s’agit d’un endpoint de compatibilité legacy. Il n’est pas inclus dans le catalogue actuel des modèles image recommandés. Pour les nouvelles intégrations, utilisez `/v1/images/edits` avec `gpt-image-2`, ou `/v1/images/generations` avec `operation: "image-to-image"` et un modèle image-to-image actuel.
</Note>
