---
title: "3D Generation"
description: "Create 3D assets from text or images and download completed model files from async task results."
---

3D generation is asynchronous. `POST /v1/3d/generations` creates a task; completed status responses return downloadable model asset URLs.

## Model Discovery

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \\
  -H "Authorization: Bearer sk-your-api-key"
```

Do not assume every 3D model supports both text-to-3D and image-to-3D. Check the selected model contract before sending `image`, `image_url`, `format`, `quality`, `style`, or `seed`.

## Create Request

```bash
curl https://api.aisonar.dev/v1/3d/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

Text-to-3D requires a `prompt`. Image-to-3D uses `image` or `image_url` when supported by the model. Prefer public URLs for production inputs unless the source image is private and your client intentionally uploads base64 data.

## Output Choices

- `format` can request output packaging such as `glb`, `fbx`, `obj`, or `usdz` when supported.
- `quality` values such as `draft`, `standard`, or `high` affect latency and cost when the model exposes them.
- `style` and `seed` are model-dependent; omit them unless the selected contract documents them.

## Polling Results

Use the returned `poll_url` first. If you need a fixed endpoint, use `GET /v1/tasks/{id}` with the same `id` / `task_id` from the create response.

Completed tasks return `model_url` and may include format-specific URLs such as `glb_url`, `fbx_url`, `obj_url`, or `usdz_url`. Download and cache the asset in your own product if users need repeat access.

## Common Pitfalls

- Do not expect a 3D asset in the create response; poll until terminal status.
- Do not promise every format for every model.
- Do not retry create requests after client timeout without checking whether the task already exists.
- Do not expose provider task IDs in your UI.
