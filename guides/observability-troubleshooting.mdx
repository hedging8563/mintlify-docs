This page is written for application teams operating AI Sonar in production. It focuses on public identifiers and user-safe logs, not provider routing internals.

## Public Identifiers

| Identifier | Where it appears | Use it for |
|------------|------------------|------------|
| `request_id` | Error bodies, Management API usage records, dashboard logs | Support and request-level reconciliation |
| `task_id` / `id` | Async create and status responses | Polling image, video, music, and 3D jobs |
| `poll_url` | Async create responses | Preferred status URL |
| `billing_transaction_id` | Non-streaming responses when settled, task status responses, Management API usage records, `X-Billing-Transaction-ID` header | Billing reconciliation |
| `X-Task-ID` | Async task responses | Header-level task correlation |

## What To Log

1. Log endpoint, model, status code, timestamp, and public identifiers.
2. Redact `Authorization`, API keys, signed URLs, private media URLs, and full prompts by default.
3. For async jobs, log the create response and every terminal status response.
4. For usage questions, call the Management API with the API key ID and date filters.
5. Use error `type`, `code`, and `param` from the response body instead of parsing provider text.

| Symptom | First check | Useful page |
|---------|-------------|-------------|
| `401` or `403` | API key, management token, organization access | [Authentication](/authentication) |

## Usage Reconciliation

`GET /v1/management/api-keys/{keyId}/usage` returns paginated key-level usage. It can filter by `scene`, `accessChannel`, `logicalModel`, `modelVendor`, `startDate`, and `endDate`. Use it for server-side reconciliation instead of scraping the dashboard.

```bash
curl "https://api.aisonar.dev/v1/management/api-keys/key_abc123def456/usage?page=1&limit=20&scene=video" \\
  -H "Authorization: Bearer mt-your-management-token"
```

Streaming responses may settle after the stream is sent, so the billing header can be absent even when usage is later recorded. For async media tasks, settlement can appear after terminal polling.

## Support Packet

When contacting support, include `request_id`, `task_id`, `billing_transaction_id` when present, endpoint, model, timestamp, and a sanitized request shape. Do not include API keys, private media, or full prompts unless support asks for a redacted sample.

Do not include provider names, channel IDs, routing identifiers, Redis keys, or upstream task URLs in customer logs or support tickets unless AI Sonar support explicitly asks for them.
