---
title: "Music Generation"
description: "Create songs, instrumentals, lyrics, and continuations with AI Sonar music tasks."
---

Music generation is asynchronous. `POST /v1/music/generations` creates a task and returns `id` / `task_id`, `status`, and usually `poll_url`. Final audio and metadata appear after polling.

## Model Discovery

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \\
  -H "Authorization: Bearer sk-your-api-key"
```

The current public contract exposes `suno_music` for music generation and `suno_lyrics` for lyric-only flows. Query the model catalog before production use instead of pinning provider assumptions.

## Create Request

```bash
curl https://api.aisonar.dev/v1/music/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

Use `action: "MUSIC"` for songs or instrumentals. Use `action: "LYRICS"` only with models that support lyric-only generation. Continuation fields such as `continue_clip_id` and `continue_at` are for continuation workflows and should be sent only when your product is continuing a previous clip.

## Polling Results

Use the returned `poll_url` first. If you need a fixed endpoint, use `GET /v1/tasks/{id}` with the same `id` / `task_id` from the create response.

Completed music tasks can include `audio_url`, `video_url`, `title`, `lyrics`, and model-normalized metadata. Store the final URLs in your application rather than polling repeatedly after terminal status.

## Common Pitfalls

- Do not display a final player until the task is `completed` and an `audio_url` exists.
- Do not use provider task IDs or provider fetch URLs in your app state.
- Do not mix lyric-only and full-music actions behind one UI without showing the difference to users.
- Do not recreate tasks on every page refresh; resume from stored `task_id`.
