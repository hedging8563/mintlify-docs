## AI Sonar Yönlendirmesi Nasıl Çalışır

### Yönlendirme Stratejileri

| Strateji | Davranış | En iyi kullanım |
|----------|---------|----------------|
| **PRIORITY** | En yüksek öncelikli mevcut sağlayıcıya yönlendirir | Güvenilirlik |
| **COST** | En ucuz mevcut sağlayıcıya yönlendirir | Maliyet optimizasyonu |

### Otomatik Yedekleme

```
İstek: claude-sonnet-4-6
  → Sağlayıcı A (birincil): 503 hatası
  → Sağlayıcı B (yedek): ✓ Başarılı
```

### Yerel Format Yönlendirmesi

Seçilen model ve yönlendirilmiş kanal bunu desteklediğinde, AI Sonar uyumluluğu artırmak için istekleri model ailesinin yerel API biçimi üzerinden yönlendirebilir:

| Model Ailesi | API Biçimi | Temel URL |
|-------------|------------|----------|
| Claude | Anthropic Mesajları | `https://api.aisonar.dev` |
| GPT | OpenAI Yanıtları | `https://api.aisonar.dev/v1` |
| Gemini | Gemini Yerel API | `https://api.aisonar.dev` veya `https://api.aisonar.dev/v1beta` istemci yoluna bağlı olarak |
| DeepSeek | OpenAI Sohbet | `https://api.aisonar.dev/v1` |

Claude Code güçlü bir Anthropic-native yoldur. Gemini-native API’ler de vardır, ancak Gemini CLI’nin kendisi hâlâ best-effort bir uyumluluk yoludur ve uzun vadeli olarak kararlı bir iş akışı gibi sunulmamalıdır.
