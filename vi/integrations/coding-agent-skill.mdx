<Note>
Kỹ năng này hướng dẫn coding agent của bạn cách tích hợp **hàng trăm API AI** của AI Sonar chỉ bằng cách mô tả những gì bạn cần. Nó được đóng gói thành một kỹ năng dùng chung agent-first và hoạt động với bất kỳ coding agent nào hỗ trợ file skill hoặc rules.

API là **[Agent-First](/guides/agent-first-api)** — ngay cả khi agent đoán sai tên model, phản hồi lỗi sẽ cho biết chính xác cách khắc phục.
</Note>

<Note>
Trang này mô tả một lớp đóng gói skill có thể tái sử dụng, chứ không phải một bề mặt tương thích giao thức. Nếu bạn cần thiết lập điểm cuối, SDK hoặc client, hãy dùng các trang tích hợp chuyên biệt cho từng công cụ.
</Note>

## Cách hoạt động

1. **Bạn mô tả những gì bạn cần** — "Tôi muốn sử dụng GPT-4 trong dự án Python của mình"
2. **Agent gọi API** — nó có thể đoán tên model hoặc điểm cuối
3. **Nếu có bất kỳ lỗi nào xảy ra** — API trả về các gợi ý có cấu trúc (`did_you_mean`, `suggestions`, `retry_after`)
4. **Agent tự sửa lỗi** — đọc các gợi ý và thử lại với các tham số đúng
5. **Bạn nhận được code hoạt động** — thường là ở lần thử đầu tiên hoặc lần thứ hai

## Cài đặt

<Note>
Sử dụng lệnh cài đặt không tương tác chính thức:

```bash
npx skills add https://github.com/hedging8563/aisonar-skills --skill aisonar-api-integration -y
```

Lệnh này cài đặt kỹ năng dùng chung `aisonar-api-integration` từ repository skills của AI Sonar.

Nếu công cụ của bạn không hỗ trợ trình cài đặt, hãy sao chép thư mục `skills/aisonar-api-integration/` từ repository vào thư mục skills hoặc rules dùng chung của công cụ.
</Note>

### Xác minh cài đặt

Hỏi coding agent của bạn:
```
Có những skills nào khả dụng?
```

Nếu bạn thấy `aisonar-api-integration`, việc cài đặt đã thành công.

</Tip>

## Ví dụ

### Chat Completion

**Bạn nói:**
```
Tôi muốn sử dụng GPT-4 trong dự án Python của mình
```

**Agent tạo ra:**
```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["TOKENLAB_API_KEY"],
    base_url="https://api.aisonar.dev/v1"
)

response = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```

Chạy bằng:

```bash
export TOKENLAB_API_KEY="sk-your-api-key"
python app.py
```

Nếu agent đoán sai tên model (ví dụ: `gpt5`), API sẽ trả về:
```json
{
  "error": {
    "code": "model_not_found",
    "did_you_mean": "gpt-5.4",
    "suggestions": [{"id": "gpt-5.4"}, {"id": "gpt-5-mini"}],
    "hint": "Did you mean 'gpt-5.4'? Use GET https://api.aisonar.dev/v1/models to list all available models."
  }
}
```

Agent đọc `did_you_mean`, sửa tên model và thử lại — tất cả đều tự động.

### Tạo ảnh

**Bạn nói:**
```
Generate images with Flux in Node.js
```

Agent gọi image API, xử lý lỗi và tạo code hoàn chỉnh với logic tải xuống và lưu tệp.

### Nhận dạng giọng nói

**Bạn nói:**
```
Integrate speech-to-text in my app
```

Agent tạo code với điểm cuối `/v1/audio/transcriptions` chính xác và xử lý file âm thanh.

## Khôi phục lỗi Agent-First

Mỗi phản hồi lỗi đều bao gồm các gợi ý có cấu trúc mà bất kỳ coding agent nào cũng có thể phân tích. Xem [tài liệu tham chiếu Agent-First API đầy đủ](/guides/agent-first-api) để biết chi tiết.

| Lỗi | API trả về gì | Agent làm gì |
|-------|---------------------|---------------------|
| Sai tên model | `did_you_mean` + `suggestions` | Tự động sửa và thử lại |
| Số dư không đủ | `balance_usd` + `suggestions` rẻ hơn | Chuyển sang model có chi phí phù hợp |
| Model không khả dụng | `alternatives` + `retry_after` | Chuyển sang model khả dụng |
| Bị giới hạn tốc độ | `retry_after` (số giây chính xác) | Chờ và thử lại |
| Context quá dài | `suggestions` với các model lớn hơn | Chuyển sang model có context lớn hơn |
| Không khớp hợp đồng cho request không phải chat | `supported_operations`, `supported_parameters`, `request_endpoint`, `recommended_request` | Dựng lại request từ hợp đồng công khai của model |

### Tối ưu hóa điểm cuối native

Khi bạn gọi `/v1/chat/completions` với model Claude hoặc Gemini, API sẽ trả về các header tối ưu hóa:



Hãy xem các header này là gợi ý định tuyến, không phải một chuyển đổi tự động bắt buộc. Lệnh gọi Claude native dùng `/v1/messages`; lệnh gọi Gemini native nên theo endpoint do the native endpoint hint header trả về, thường là `/v1beta/gemini`.

### Khám phá model

Không cần tìm kiếm trong tài liệu:

```bash
# Tổng quan API máy có thể đọc
curl https://api.aisonar.dev/llms.txt

# List models by category
curl "https://api.aisonar.dev/v1/models?category=chat" -H "Authorization: Bearer sk-KEY"
curl "https://api.aisonar.dev/v1/models?category=image" -H "Authorization: Bearer sk-KEY"

# Danh sách đề xuất cho kịch bản không phải chat
curl "https://api.aisonar.dev/v1/models?recommended_for=image" -H "Authorization: Bearer sk-KEY"
curl "https://api.aisonar.dev/v1/models?recommended_for=translation" -H "Authorization: Bearer sk-KEY"

# Đọc hợp đồng công khai của một model trước khi thử lại request không phải chat
curl "https://api.aisonar.dev/v1/models/gpt-image-2" -H "Authorization: Bearer sk-KEY"

# Lọc theo năng lực
curl "https://api.aisonar.dev/v1/models?tag=coding" -H "Authorization: Bearer sk-KEY"
```
