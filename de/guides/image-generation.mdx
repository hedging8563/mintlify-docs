---
title: "Bildgenerierung"
description: "Erstellen Sie Text-zu-Bild-, Bild-zu-Bild- und Bildbearbeitungs-Workflows gegen die öffentlichen Bildverträge von AI Sonar."
---

Die AI Sonar-Bild-APIs decken mehrere Modellfamilien mit unterschiedlichen Anforderungsformaten ab. Verwenden Sie den öffentlichen Modellvertrag und die spezifische API-Referenz für den Endpunkt, bevor Sie davon ausgehen, dass ein Feld in jedem Bildmodell funktioniert.

## Wählen Sie den Endpunkt

| Ziel | Endpunkt | Hinweise |
|------|----------|-------|
| Text-zu-Bild | `POST /v1/images/generations` | Verwenden Sie `prompt` und ein explizites `model` |
| Bild-zu-Bild / Referenzbild | `POST /v1/images/generations` | Verwenden Sie `operation: "image-to-image"` plus `image_url`, `image_urls` oder `reference_image_urls`, wenn das Modell dies unterstützt |
| GPT Bildbearbeitung | `POST /v1/images/edits` | Unterstützt multipart `image`, JSON `image_url` / `image_urls` und offizielle `images[]` Referenzen für unterstützte Bearbeitungsmodelle |
| Legacy-Variation | `POST /v1/images/variations` | Bevorzugen Sie aktuelle Generierungs- oder Bearbeitungsrouten für neue Arbeiten |
| Async-Status | `GET /v1/tasks/{id}` | Verwenden Sie zuerst `poll_url`, wenn vorhanden |

## Modellerkennung

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=image" \\
  -H "Authorization: Bearer sk-your-api-key"
```

Die Bildmodellparameter unterscheiden sich stark. Lesen Sie den ausgewählten Modellvertrag, bevor Sie `size`, `quality`, `aspect_ratio`, `resolution`, `async`, `background` oder `output_format` senden.

## Eingaben und modell-spezifische Felder

- `gpt-image-2` verwendet OpenAI-ähnliche `size` und `quality`; lassen Sie `size` oder `quality` weg, damit AI Sonar `auto` verwenden kann, wo es unterstützt wird.
- Die Bildfamilien Gemini und Nano Banana verwenden im Allgemeinen `aspect_ratio`; senden Sie `resolution` nur, wenn das ausgewählte Modell es bereitstellt.
- Anfragen für Referenzbilder von Nano Banana gehören zu `/v1/images/generations` mit `operation: "image-to-image"` und `image_urls`, nicht zu `/v1/images/edits`.
- `/v1/images/generations` akzeptiert keine obersten `images[]` oder `file_id`; das sind Formen des Bearbeitungsflusses.
- Remote-Bild-URLs müssen öffentliche `http`/`https` URLs sein. Vermeiden Sie private Netzwerke, eingebettete Anmeldeinformationen, Fragmente und kurzlebige signierte URLs.

## Anfrage erstellen

```bash
curl https://api.aisonar.dev/v1/images/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-image-2",
    "prompt": "Ein sauberes Produktfoto einer keramischen Kaffeetasse auf einem Walnuss-Schreibtisch",
    "size": "1024x1024",
    "response_format": "url"
  }'
```

## Ergebnisse abfragen

Einige Bildrouten geben inline `data[].url` oder `data[].b64_json` zurück. Andere Routen geben `status: "pending"`, `task_id` und `poll_url` zurück. Wenn Sie die asynchrone Bildgenerierung oder -bearbeitung anfordern, geben die abgeschlossenen Aufgaben Ergebnisse in Form von URLs zurück; verwenden Sie synchrone Anfragen, wenn Sie speziell `b64_json` benötigen.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \\
  -H "Authorization: Bearer sk-your-api-key"
```

## Häufige Fallstricke

- Gehen Sie nicht davon aus, dass jedes Bildmodell synchron ist.
- Fordern Sie `b64_json` nicht von asynchronen Bildaufgaben an; asynchrone Ergebnisse sind URL-orientiert.
- Senden Sie `resolution` nicht an bildverhältnisbasierte Gemini-Bildmodelle.
- Verwenden Sie `/v1/images/edits` nicht für Nano Banana Bild-zu-Bild-Flüsse.
- Setzen Sie HTTP-Client-Timeouts auf mindestens `120s` für synchrone hochauflösende Bildanfragen.
