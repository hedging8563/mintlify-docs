## Empfohlener Ablauf

1. Rufe [Modelle auflisten](/de/api-reference/models/list-models) mit `recommended_for=music` auf.
2. Nutze [Musik erstellen](/de/api-reference/music/create-music), um einen Task mit `prompt` und optional `title`, `tags`, `action`, `mv` oder Fortsetzungsparametern zu erstellen.
3. Speichere die zurückgegebenen Werte `id` / `task_id` und `poll_url`.
4. Frage ab, bis der Task `completed` oder `failed` ist, und lies dann `audio_url`, optional `video_url`, `title` und `lyrics`.

## Modelle finden

Musikgenerierung ist asynchron. Baue nicht gegen Upstream-Anbieternamen oder Provider-Task-IDs; AI Sonar gibt eine normalisierte Task-Identität und Polling-URL zurück.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Musik-Task erstellen

`action: "MUSIC"` erstellt je nach Prompt und Modell Songs oder Instrumentals. `action: "LYRICS"` ist für reine Lyrics-Flows, wenn das gewählte Modell dies unterstützt. `continue_clip_id` und `continue_at` verwendest du nur für Fortsetzungsflows, die der Modellvertrag dokumentiert.

```bash
curl https://api.aisonar.dev/v1/music/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

## Ergebnisse pollen

Bevorzuge immer die zurückgegebene `poll_url`. Der stabile Fallback ist [Task-Status abrufen](/de/api-reference/tasks/get-task-status). Abgeschlossene Musik-Tasks können `audio_url`, `video_url`, `title`, `lyrics` und modellnormalisierte Metadaten enthalten.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
