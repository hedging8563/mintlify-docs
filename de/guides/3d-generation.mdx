title: "3D-Generierung"
description: "Erstelle 3D-Assets aus Text oder Bildern mit den AI Sonar-3D-APIs und asynchronem Task-Polling."
---
## Empfohlener Ablauf

1. Rufe [Modelle auflisten](/de/api-reference/models/list-models) mit `recommended_for=3d` auf.
2. Nutze [3D erstellen](/de/api-reference/3d/create-3d) mit einem Text-`prompt`; füge `image` oder `image_url` nur für Modelle hinzu, die Bild-zu-3D unterstützen.
3. Wähle optionale Felder wie `format`, `quality`, `style` oder `seed` gemäß dem Vertrag des gewählten Modells.
4. Frage `poll_url` ab, bis das Asset bereit ist, und lade dann `model_url` oder formatbezogene URLs wie `glb_url` / `fbx_url` herunter.

## Modelle finden

3D-Generierung ist taskbasiert. Behandle den Modellkatalog als Quelle der Wahrheit, statt einen einzelnen Provider oder ein festes Modell anzunehmen.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \
  -H "Authorization: Bearer sk-your-api-key"
```

## 3D-Task erstellen

Text-zu-3D-Requests benötigen `prompt`. Bild-zu-3D-Requests fügen entweder `image` als base64-Daten oder `image_url` hinzu. Nutze `format` und `quality` nur, wenn der Modellvertrag sie anbietet.

```bash
curl https://api.aisonar.dev/v1/3d/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

## Ergebnisse pollen

Verwende zuerst die zurückgegebene `poll_url`. Der stabile Fallback ist [Task-Status abrufen](/de/api-reference/tasks/get-task-status). Abgeschlossene 3D-Tasks liefern `model_url` und können formatbezogene Felder wie `glb_url` oder `fbx_url` enthalten.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
