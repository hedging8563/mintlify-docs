## Überblick

Die meisten allgemeinen AI-Chat-Apps verbinden sich über den standardmäßigen OpenAI-kompatiblen Chat-Pfad mit AI Sonar.

In der Praxis bedeutet das:
- Verwenden Sie `https://api.aisonar.dev/v1` als Base URL, sofern die App nicht ausdrücklich nach einer vollständigen Endpunkt-URL fragt
- Verwenden Sie Ihren AI Sonar API-Schlüssel als API-Schlüssel im OpenAI-Stil
- Wählen Sie Modelle aus, die die App in einem standardmäßigen Chat-Ablauf verwenden kann

<Note>
Allgemeine Chat-Apps sind in der Regel **auf Chat Completions ausgerichtet** und keine Codex-Responses-WebSocket-Clients. Für Codex CLI/App, Cursor oder andere Coding-Tools, die Responses-spezifisches Verhalten benötigen, verwenden Sie stattdessen die dedizierten Integrationsleitfäden.
</Note>

<Note>
**Typ**: Chat-App

**Primärer Pfad**: OpenAI-kompatibler Chat-Completions-Pfad

**Support-Niveau**: Unterstützter Pfad
</Note>

## Unterstützte Anwendungen

| Anwendung | Plattform | Hinweise |
|-------------|----------|-------|
| Open WebUI | Web, selbst betrieben | Beliebter selbst betriebener Client mit OpenAI-kompatiblen Verbindungen |
| LibreChat | Web, selbst betrieben | Mehranbieter-Chat-Oberfläche mit Unterstützung für benutzerdefinierte OpenAI-Endpunkte |
| LobeChat | Web, Desktop, selbst betrieben | Starke Unterstützung für benutzerdefinierte Anbieter |
| Cherry Studio | Windows, macOS, Linux | Guter Anbieter-Ablauf für OpenAI-kompatible Verbindungen |
| Chatbox | Windows, macOS, Linux | Einfache OpenAI-kompatible Einrichtung |
| TypingMind | Web, Desktop | Verwendet einen benutzerdefinierten Endpunkt-Stil |
| NextChat | Web, selbst betrieben | Leichter ChatGPT-Stil-Client mit Base-URL-Überschreibungen |
| ChatWise | Windows, macOS | Standardmuster für benutzerdefinierte Anbieter |

## Standard-Verbindungswerte

| Feld | Wert |
|-------|-------|
| Basis-URL | `https://api.aisonar.dev/v1` |
| API-Schlüssel | `sk-your-api-key` |
| API-Typ | OpenAI-kompatibel |

Wenn eine App statt einer Base URL eine **vollständige Endpunkt-URL** verlangt, verwenden Sie:
- Chat Completions: `https://api.aisonar.dev/v1/chat/completions`

## Beispiel-App-Einrichtungen

<Tabs>
  <Tab title="LobeChat">
    **Selbst betriebenes LobeChat**

    ```bash
    OPENAI_API_KEY=sk-your-api-key
    OPENAI_PROXY_URL=https://api.aisonar.dev/v1
    ```

    **LobeChat Cloud**
    1. Öffnen Sie **Settings** → **AI Service Provider**
    2. Wählen Sie **OpenAI**
    3. Geben Sie Ihren AI Sonar API-Schlüssel ein
    4. Setzen Sie den benutzerdefinierten Endpunkt auf `https://api.aisonar.dev/v1`
  </Tab>

  <Tab title="Cherry Studio">
    1. Öffnen Sie **Settings** → **Model Provider**
    2. Fügen Sie einen **OpenAI-kompatibel** Anbieter hinzu
    3. Geben Sie ein:
       - **Name**: AI Sonar
       - **API Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
    4. Fügen Sie die Modelle hinzu, die Sie verwenden möchten
  </Tab>

  <Tab title="Chatbox">
    1. Öffnen Sie **Settings** → **AI Model**
    2. Wählen Sie **OpenAI API Compatible**
    3. Geben Sie ein:
       - **API Host**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
       - **Model**: `gpt-5-mini` oder `gpt-4o`
  </Tab>

  <Tab title="TypingMind">
    1. Öffnen Sie **Settings**
    2. Wählen Sie **Custom Endpoint**
    3. Geben Sie ein:
       - **Endpunkt-URL**: `https://api.aisonar.dev/v1/chat/completions`
       - **API Key**: `sk-your-api-key`
    4. Konfigurieren Sie die Modelle, die Sie bereitstellen möchten
  </Tab>

  <Tab title="Open WebUI">
    1. Öffnen Sie **Admin Settings** → **Connections**
    2. Fügen Sie eine **OpenAI-kompatible** Verbindung hinzu
    3. Geben Sie ein:
       - **Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
    4. Speichern Sie und wählen Sie die Modelle aus, die Sie freigeben möchten
  </Tab>

  <Tab title="LibreChat">
    Fügen Sie in der LibreChat-Anbieterkonfiguration einen OpenAI-kompatiblen Endpunkt hinzu und verweisen Sie auf:

    ```yaml
    customEndpoints:
      - name: AI Sonar
        apiKey: '${OPENAI_API_KEY}'
        baseURL: 'https://api.aisonar.dev/v1'
        models:
          default: ['gpt-5-mini', 'gpt-5.4', 'claude-sonnet-4-6']
          fetch: false
    ```
  </Tab>
</Tabs>

## Tipps

<AccordionGroup>
  <Accordion title="Base URL bevorzugen, wenn die App sie unterstützt">
    Die meisten Apps lassen sich einfacher konfigurieren und sind weniger fehleranfällig, wenn Sie `https://api.aisonar.dev/v1` als Base URL statt einer vollständigen Endpunkt-URL angeben.
  </Accordion>

  <Accordion title="Zuerst kleinere Modelle verwenden">
    Beginnen Sie für alltägliche Chats mit `gpt-5-mini` oder `gpt-4o` und verlagern Sie aufwendigere Reasoning-Aufgaben anschließend auf `gpt-5.4` oder `claude-opus-4-6`.
  </Accordion>

  <Accordion title="Coding-Tools getrennt halten">
    Allgemeine Chat-Apps und Coding-Agents benötigen oft unterschiedliches Endpunkt-Verhalten. Lassen Sie Tools im Stil von Codex/Cursor auf ihren dedizierten Integrationsseiten.
  </Accordion>
</AccordionGroup>

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="Verbindungsfehler">
    - Vergewissern Sie sich, dass die Base URL exakt `https://api.aisonar.dev/v1` ist
    - Entfernen Sie abschließende Schrägstriche, wenn die App Pfade automatisch anhängt
    - Wenn die App eine vollständige Endpunkt-URL verlangt, verwenden Sie `https://api.aisonar.dev/v1/chat/completions`
  </Accordion>

  <Accordion title="401 Unauthorized">
    - Überprüfen Sie Ihren API-Schlüssel erneut
    - Bestätigen Sie, dass der Schlüssel im AI Sonar-Dashboard aktiv ist
    - Stellen Sie sicher, dass keine zusätzlichen Leerzeichen oder Zeilenumbrüche vorhanden sind
  </Accordion>

  <Accordion title="Modell nicht verfügbar">
    - Überprüfen Sie den Modellnamen exakt
    - Versuchen Sie einen breit unterstützten Standard wie `gpt-5-mini`, `gpt-4o` oder `claude-sonnet-4-6`
    - Prüfen Sie die aktuelle Verfügbarkeit in den AI Sonar-Dokumenten oder im Dashboard
  </Accordion>
</AccordionGroup>
