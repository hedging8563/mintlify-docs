## Übersicht

Dieser Endpunkt bleibt für legacy OpenAI-kompatible Variations-Clients erhalten. Für aktuelle Image-to-Image-Workflows ist er nicht empfohlen; verwenden Sie nach Möglichkeit `/v1/images/edits` mit `gpt-image-2` oder `/v1/images/generations` mit einem Image-to-Image-Modell wie `nano-banana-pro`. Erfordert `multipart/form-data`.

## Anfragekörper

**Timeout für synchrone Anfragen:** Einige geroutete Bildanbieter geben das endgültige Bild inline zurück und warten dafür, bis die Generierung abgeschlossen ist. Hochauflösende oder hochwertige Anfragen können fast eine Minute oder länger dauern; setzen Sie das Timeout Ihres HTTP-Clients daher auf mindestens `120s`. Wenn die Create-Antwort `status: "pending"`, `task_id` oder `poll_url` enthält, folgen Sie stattdessen der zurückgegebenen `poll_url`.

<ParamField body="image" type="file" required>
  Das Bild als Basis für die Variation. Muss eine gültige PNG-Datei sein, kleiner als 50MB und quadratisch.
</ParamField>

<ParamField body="model" type="string" required>
  Erforderliche Modell-ID für legacy variationskompatible Routen. Es gibt keinen impliziten Standard; für aktuelle Image-to-Image-Workflows bevorzugen Sie `/v1/images/edits` oder `/v1/images/generations` mit einem aktuellen Modell.
</ParamField>

<ParamField body="n" type="integer" default="1">
  Die Anzahl der zu generierenden Bilder. Muss zwischen 1 und 10 liegen.
</ParamField>

<ParamField body="size" type="string">
  Die Größe der erzeugten Bilder für legacy variationskompatible Routen. Unterstützte Werte hängen vom ausgewählten Modell ab.
</ParamField>

<ParamField body="response_format" type="string" default="url">
  Das Format der zurückgegebenen Bilder. Muss `url` oder `b64_json` sein.
</ParamField>

<ParamField body="user" type="string">
  Eine eindeutige Kennung für Ihren Endbenutzer zur Missbrauchsüberwachung.
</ParamField>

## Antwort

<ResponseField name="created" type="integer">
  Unix-Zeitstempel der Bilderstellung.
</ResponseField>

<ResponseField name="data" type="array">
  Array der generierten Bildvariationen.

  Jedes Objekt enthält:
  - `url` (string): URL des Variationsbildes (wenn response_format `url` ist)
  - `b64_json` (string): Base64-kodiertes Bild (wenn response_format `b64_json` ist)
</ResponseField>

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.aisonar.dev/v1"
)

response = client.images.create_variation(
    model="your-legacy-variation-model",
    image=open("cat.png", "rb"),
    n=2,
    size="1024x1024"
)

for image in response.data:
    print(image.url)
```

```go Go
package main

import (
    "bytes"
    "fmt"
    "io"
    "mime/multipart"
    "net/http"
    "os"
)

func main() {
    body := &bytes.Buffer{}
    writer := multipart.NewWriter(body)

    writer.WriteField("model", "your-legacy-variation-model")

    file, _ := os.Open("cat.png")
    defer file.Close()
    part, _ := writer.CreateFormFile("image", "cat.png")
    io.Copy(part, file)

    writer.WriteField("n", "2")
    writer.WriteField("size", "1024x1024")
    writer.Close()

    req, _ := http.NewRequest("POST", "https://api.aisonar.dev/v1/images/variations", body)
    req.Header.Set("Authorization", "Bearer sk-your-api-key")
    req.Header.Set("Content-Type", writer.FormDataContentType())

    client := &http.Client{}
    resp, _ := client.Do(req)
    defer resp.Body.Close()

    result, _ := io.ReadAll(resp.Body)
    fmt.Println(string(result))
}
```

## Hinweise

<Note>
  Dies ist ein legacy Kompatibilitätsendpunkt. Er ist nicht im aktuellen empfohlenen Bildmodellkatalog enthalten. Für neue Integrationen verwenden Sie `/v1/images/edits` mit `gpt-image-2` oder `/v1/images/generations` mit `operation: "image-to-image"` und einem aktuellen Image-to-Image-Modell.
</Note>
