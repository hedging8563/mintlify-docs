## Übersicht

Dieser Endpunkt dient in erster Linie den Dateiobjekten von AI Sonars OpenAI-kompatibler Batch-API.

Wenn die Anfrage in den Anthropic-Files-Kompatibilitätsmodus wechselt, kann dieselbe Route außerdem Anthropic-ähnliche Datei-Metadaten für den gebundenen Workspace zurückgeben.

## Hinweise

- Batch-Eingabedateien müssen `purpose=batch` verwenden.
- Verwenden Sie `purpose=batch_output`, um sich auf erzeugte Ausgabedateien zu konzentrieren.
- `completion_window` beträgt derzeit `24h`.
- Die Ausgabe-Reihenfolge ist nicht garantiert; immer nach `custom_id` abgleichen.
- Streaming wird innerhalb von Batch-Elementen nicht unterstützt.

## Beispiel

<RequestExample>
```bash cURL
curl -X GET "https://api.aisonar.dev/v1/files/file_abc123" \
  -H "Authorization: Bearer sk-your-api-key"
```
</RequestExample>

## Anfrage / Antwort

Verwenden Sie das interaktive OpenAPI-Panel oben für das genaue Schema.

## Betriebliche Tipps

- Verwenden Sie `custom_id` für idempotente nachgelagerte Abstimmungen.
- Erwarten Sie `output_file_id` und `error_file_id` erst, nachdem der Worker den Batch abgeschlossen hat.
- Die Preise für Batches können von den Preisen für synchrone Anfragen abweichen, da die Rabattregeln für `isBatchRequest=true` gelten.

</ResponseExample>

<ResponseField name="id" type="string">Dateikennung für verwandte APIs.</ResponseField>
<ResponseField name="filename" type="string">Kennung für Folgeaufrufe und Support.</ResponseField>
<ResponseField name="purpose" type="string">Von der API zurückgegebener Event- oder Nachrichtentyp.</ResponseField>
