## Übersicht

Dieser Endpunkt ist Teil von AI Sonar's OpenAI-kompatiblem Batch-API. Batch-Jobs verwenden hochgeladene JSONL-Dateien, laufen asynchron und können später Ausgabedateien/Fehlerdateien zurückgeben.

## Hinweise

- Batch-Eingabedateien müssen `purpose=batch` verwenden.
- `completion_window` beträgt derzeit `24h`.
- Die Ausgabereihenfolge ist nicht garantiert; immer nach `custom_id` abgleichen.
- Streaming wird innerhalb von Batch-Elementen nicht unterstützt.

## Beispiel

<RequestExample>
```bash cURL
curl -X POST "https://api.aisonar.dev/v1/batches/batch_abc123/cancel" \
  -H "Authorization: Bearer sk-your-api-key"
```
</RequestExample>

## Betriebliche Tipps

- Verwenden Sie `custom_id` für idempotente nachgelagerte Abstimmungen.
- Erwarten Sie `output_file_id` und `error_file_id` erst, nachdem der Worker den Batch abgeschlossen hat.
- Die Preisgestaltung für Batches kann von der Preisgestaltung für synchrone Anfragen abweichen, da die Rabattregeln für `isBatchRequest=true` gelten.
