---
title: "画像生成"
description: "AI Sonarの公開画像契約に対して、テキストから画像、画像から画像、画像編集のワークフローを構築します。"
---

AI Sonarの画像APIは、異なるリクエスト形式を持ついくつかのモデルファミリーをカバーしています。すべての画像モデルでフィールドが機能することを前提とする前に、公開モデル契約とエンドポイント固有のAPIリファレンスを使用してください。

## エンドポイントの選択

| 目標 | エンドポイント | メモ |
|------|----------------|------|
| テキストから画像 | `POST /v1/images/generations` | `prompt`と明示的な`model`を使用 |
| 画像から画像 / 参照画像 | `POST /v1/images/generations` | モデルがサポートしている場合は、`operation: "image-to-image"`に加えて`image_url`、`image_urls`、または`reference_image_urls`を使用 |
| GPT画像編集 | `POST /v1/images/edits` | サポートされている編集モデルのために、マルチパート`image`、JSON形式の`image_url` / `image_urls`、および公式の`images[]`参照をサポート |
| レガシー変種 | `POST /v1/images/variations` | 新しい作業には現在の生成または編集ルートを優先 |
| 非同期ステータス | `GET /v1/tasks/{id}` | 存在する場合は最初に`poll_url`を使用 |

## モデル発見

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=image" \\
  -H "Authorization: Bearer sk-your-api-key"
```

画像モデルのパラメータは大きく異なります。`size`、`quality`、`aspect_ratio`、`resolution`、`async`、`background`、または`output_format`を送信する前に、選択したモデル契約を読んでください。

## 入力とモデル固有のフィールド

- `gpt-image-2`はOpenAIスタイルの`size`と`quality`を使用します。AI Sonarがサポートする場合は、`size`または`quality`を省略して`auto`を使用させてください。
- GeminiおよびNano Banana画像ファミリーは一般的に`aspect_ratio`を使用します。選択したモデルがそれを公開している場合にのみ`resolution`を送信してください。
- Nano Bananaの参照画像リクエストは、`operation: "image-to-image"`と`image_urls`を持つ`/v1/images/generations`に属し、`/v1/images/edits`には属しません。
- `/v1/images/generations`はトップレベルの`images[]`または`file_id`を受け付けません。これらは編集フローの形式です。
- リモート画像URLは公開の`http`/`https` URLでなければなりません。プライベートネットワーク、埋め込まれた資格情報、フラグメント、および短命の署名付きURLは避けてください。

## リクエストの作成

```bash
curl https://api.aisonar.dev/v1/images/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-image-2",
    "prompt": "クリーンな製品写真、ウォールナットのデスクの上にあるセラミックコーヒーカップ",
    "size": "1024x1024",
    "response_format": "url"
  }'
```

## 結果のポーリング

一部の画像ルートはインラインの`data[].url`または`data[].b64_json`を返します。他のルートは`status: "pending"`、`task_id`、および`poll_url`を返します。非同期の画像生成または編集をリクエストした場合、完了したタスクの結果はURLを返します。特に`b64_json`が必要な場合は同期リクエストを使用してください。

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \\
  -H "Authorization: Bearer sk-your-api-key"
```

## よくある落とし穴

- すべての画像モデルが同期であるとは限りません。
- 非同期画像タスクから`b64_json`をリクエストしないでください。非同期結果はURL指向です。
- 比率のみのGemini画像モデルに`resolution`を送信しないでください。
- Nano Bananaの画像から画像へのフローに`/v1/images/edits`を使用しないでください。
- 同期の高解像度画像リクエストには、HTTPクライアントのタイムアウトを少なくとも`120s`に設定してください。
