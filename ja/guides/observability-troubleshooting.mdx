このページは、AI Sonarを本番環境で運用するアプリケーションチーム向けに書かれています。プロバイダーのルーティング内部ではなく、公開識別子とユーザー安全なログに焦点を当てています。

## 公開識別子

| 識別子 | 出現場所 | 使用目的 |
|--------|----------|----------|
| `request_id` | エラーボディ、管理APIの使用記録、ダッシュボードログ | サポートおよびリクエストレベルの調整 |
| `task_id` / `id` | 非同期作成およびステータス応答 | 画像、動画、音楽、3Dジョブのポーリング |
| `poll_url` | 非同期作成応答 | 優先ステータスURL |
| `billing_transaction_id` | 決済時の非ストリーミング応答、タスクステータス応答、管理APIの使用記録、`X-Billing-Transaction-ID`ヘッダー | 請求の調整 |
| `X-Task-ID` | 非同期タスク応答 | ヘッダーレベルのタスク相関 |

## ログに記録する内容

1. エンドポイント、モデル、ステータスコード、タイムスタンプ、および公開識別子をログに記録します。
2. デフォルトで`Authorization`、APIキー、署名付きURL、プライベートメディアURL、および完全なプロンプトを削除します。
3. 非同期ジョブの場合、作成応答とすべての端末ステータス応答をログに記録します。
4. 使用に関する質問については、APIキーIDと日付フィルターを使用して管理APIを呼び出します。
5. プロバイダーのテキストを解析する代わりに、応答ボディからエラー`type`、`code`、および`param`を使用します。

## トラブルシューティングチェックリスト

| 症状 | 最初に確認すること | 有用なページ |
|------|------------------|--------------|
| `401` または `403` | APIキー、管理トークン、組織アクセス | [認証](/ja/authentication) |
| `429` | レート制限の階層とクライアントの再試行動作 | [レート制限](/ja/guides/rate-limits) |
| `400 invalid_request_error` | サポートされていないフィールド、間違ったエンドポイント、またはモデル固有の契約不一致 | [エラーハンドリング](/ja/guides/error-handling) |
| 非同期タスクが終了しない | `poll_url`をポーリングし、最終`status`を確認し、重複タスクの作成を避ける | [非同期ジョブとポーリング](/ja/guides/async-jobs-polling) |
| コストがUIと一致しない | 使用記録内の`request_id`と`billing_transaction_id`を比較する | [請求と価格設定](/ja/guides/billing) |

## 使用の調整

`GET /v1/management/api-keys/{keyId}/usage`は、ページネーションされたキーごとの使用状況を返します。`scene`、`accessChannel`、`logicalModel`、`modelVendor`、`startDate`、および`endDate`でフィルタリングできます。ダッシュボードをスクレイピングする代わりに、サーバーサイドの調整に使用します。

```bash
curl "https://api.aisonar.dev/v1/management/api-keys/key_abc123def456/usage?page=1&limit=20&scene=video" \\
  -H "Authorization: Bearer mt-your-management-token"
```

ストリーミング応答はストリームが送信された後に決済される場合があるため、使用が後で記録される場合でも請求ヘッダーが存在しないことがあります。非同期メディアタスクの場合、決済は端末ポーリングの後に表示されることがあります。

## サポートパケット

サポートに連絡する際は、`request_id`、`task_id`、`billing_transaction_id`（存在する場合）、エンドポイント、モデル、タイムスタンプ、およびサニタイズされたリクエスト形状を含めてください。サポートが削除されたサンプルを要求しない限り、APIキー、プライベートメディア、または完全なプロンプトを含めないでください。

AI Sonarのサポートが明示的に要求しない限り、顧客ログやサポートチケットにプロバイダー名、チャネルID、ルーティング識別子、Redisキー、または上流タスクのURLを含めないでください。
