<Note>
このガイドは、**self-hosted OpenClaw** ユーザー向けで、AI Sonar を AI プロバイダとして接続したい方向けです。
</Note>

## 概要

現在の OpenClaw バージョンでは、推奨される方法は `models.providers` 経由で AI Sonar を構成することです。

素早く始めたいだけなら、`aisonar` のみを構成するだけで十分です。**Responses API**、**Claude native**、**Gemini native**、または **MiniMax native** の動作が明示的に必要な場合にのみ、他のプロバイダを追加してください。

| プロバイダー | OpenClaw `api` | 最適用途 | baseUrl |
|----------|----------------|----------|---------|
| `aisonar` | `openai-completions` | GPT、DeepSeek、Qwen、およびほとんどの OpenAI 互換コール | `https://api.aisonar.dev/v1` |
| `aisonar-responses` | `openai-responses` | `/v1/responses` セマンティクスを期待する OpenAI Responses ワークフロー | `https://api.aisonar.dev/v1` |
| `aisonar-claude` | `anthropic-messages` | Claude のネイティブ Messages API | `https://api.aisonar.dev` |
| `aisonar-gemini` | `google-generative-ai` | Gemini のネイティブ API 形式 | `https://api.aisonar.dev` |
| `aisonar-minimax` | `anthropic-messages` | MiniMax のネイティブルーティング | `https://api.aisonar.dev` |

<Warning>
`/v1` サフィックスは `openai-completions` と `openai-responses` のみで使用してください。

`anthropic-messages` や `google-generative-ai` のようなネイティブプロバイダは、`/v1` なしの `https://api.aisonar.dev` を使用する必要があります。さもないと OpenClaw が誤ったアップストリームパスを構築する可能性があります。
</Warning>

## 前提条件

- セルフホストの OpenClaw インスタンス
- AI Sonar API Key — [Get one here](https://console.aisonar.dev)

## 設定

OpenClaw の設定を編集します:

- **セルフホスト**: `~/.openclaw/openclaw.json`

`models.providers` の下に AI Sonar プロバイダを追加します:

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "aisonar-claude/claude-sonnet-4-6"
      }
    }
  },
  models: {
    mode: "merge",
    providers: {
      aisonar: {
        api: "openai-completions",
        baseUrl: "https://api.aisonar.dev/v1",
        apiKey: "sk-your-api-key",
        models: [
          { id: "gpt-4o", name: "GPT-4o" },
          { id: "deepseek-r1", name: "DeepSeek R1" },
          { id: "qwen3-32b", name: "Qwen 3 32B" }
        ]
      },
      "aisonar-responses": {
        api: "openai-responses",
        baseUrl: "https://api.aisonar.dev/v1",
        apiKey: "sk-your-api-key",
        models: [
          { id: "gpt-4o", name: "GPT-4o (Responses)" },
          { id: "gpt-5.2", name: "GPT-5.2 (Responses)" }
        ]
      },
      "aisonar-claude": {
        api: "anthropic-messages",
        baseUrl: "https://api.aisonar.dev",
        apiKey: "sk-your-api-key",
        models: [
          { id: "claude-sonnet-4-6", name: "Claude Sonnet 4.6" },
          { id: "claude-opus-4-6", name: "Claude Opus 4.6" }
        ]
      },
      "aisonar-gemini": {
        api: "google-generative-ai",
        baseUrl: "https://api.aisonar.dev",
        apiKey: "sk-your-api-key",
        models: [
          { id: "gemini-2.5-flash", name: "Gemini 2.5 Flash" },
          { id: "gemini-3-flash-preview", name: "Gemini 3 Flash Preview" }
        ]
      },
      "aisonar-minimax": {
        api: "anthropic-messages",
        baseUrl: "https://api.aisonar.dev",
        apiKey: "sk-your-api-key",
        models: [
          { id: "minimax-m1", name: "MiniMax M1" }
        ]
      }
    }
  }
}
```

<Info>
5 つのプロバイダはすべて **同じ API Key** を使用します。AI Sonar アカウントは 1 つあれば十分です。
</Info>

<Info>
上記の `models` 配列は一般的な例のみを示しています。必要に応じて各プロバイダにモデル ID を追加してください。
</Info>

## モデルの使用

OpenClaw は引き続き `provider/model` 形式でモデルを参照します:

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "aisonar-gemini/gemini-2.5-flash"
      }
    }
  }
}
```

### モデルの例

| プロバイダー | モデル参照 | 説明 |
|----------|-----------------|-------------|
| `aisonar` | `aisonar/gpt-4o` | OpenAI 互換ルート |
| `aisonar` | `aisonar/deepseek-r1` | DeepSeek 推論モデル |
| `aisonar-responses` | `aisonar-responses/gpt-4o` | Responses API ルート |
| `aisonar-claude` | `aisonar-claude/claude-sonnet-4-6` | ネイティブ Claude Messages ルート |
| `aisonar-gemini` | `aisonar-gemini/gemini-2.5-flash` | ネイティブ Gemini ルート |
| `aisonar-minimax` | `aisonar-minimax/minimax-m1` | ネイティブ MiniMax ルート |

利用可能なモデルの一覧は [aisonar.dev/models](https://aisonar.dev/models) を参照してください。

## どのプロバイダを使うべきか

- `aisonar`: 一般的なエージェントやチャット用途のデフォルト選択。
- `aisonar-responses`: OpenClaw のワークフローが OpenAI Responses のセマンティクスに明示的に依存する場合に使用。
- `aisonar-claude`: Claude のネイティブ Messages 動作を利用したい場合に使用。
- `aisonar-gemini`: Gemini ネイティブのリクエスト/レスポンス形式や既存の Gemini スタイル統合を使いたい場合に使用。
- `aisonar-minimax`: MiniMax をネイティブルートで利用したい場合に使用。

Gemini ネイティブの動作が不要な場合でも、OpenAI 互換ルートの `aisonar/gemini-*` 経由で Gemini モデルを呼び出すことができます。

## よくある間違い

<AccordionGroup>
  <Accordion title="古いトップレベルの providers 配列をまだ使っている">
    現在の OpenClaw ドキュメントは `models.providers` を使用します。古いトップレベルの `providers` 配列形式のままにしておくと、OpenClaw が設定を無視したり、プロバイダプレフィックスの解決に失敗したりする可能性があります。
  </Accordion>
  <Accordion title="aisonar-responses で /v1 を忘れる">
    `openai-responses` は AI Sonar の `/v1/responses` パスにマップされるため、`aisonar-responses` は `https://api.aisonar.dev/v1` を使用する必要があります。
  </Accordion>
  <Accordion title="aisonar-claude、aisonar-gemini、aisonar-minimax に /v1 を追加してしまう">
    `anthropic-messages` や `google-generative-ai` は `https://api.aisonar.dev` を `/v1` なしで使用するべきです。`/v1` を追加すると不正確なリクエストパスが生成される可能性があります。
  </Accordion>
  <Accordion title="OpenClaw はまだネイティブ Gemini をサポートしていますか？">
    はい。現在の OpenClaw ドキュメントは組み込みの `google` プロバイダを引き続き含み、`api: "google-generative-ai"` を使ったカスタムプロバイダもサポートしています。したがって `aisonar-gemini` は OpenClaw ユーザーにとって有効なネイティブ Gemini ルートのままです。
  </Accordion>
</AccordionGroup>

## 設定の検証

設定を保存したら、OpenClaw インスタンスを再起動し、簡単なメッセージでテストしてください。レスポンスが返ってくれば、プロバイダは正しく構成されています。

```bash
# セルフホスト: サービスを再起動
systemctl --user restart openclaw    # Linux
launchctl stop cc.aisonar.openclaw && launchctl start cc.aisonar.openclaw  # macOS
```
