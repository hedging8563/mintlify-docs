## 概要

ほとんどの汎用 AI チャットアプリは、標準の OpenAI 互換チャット経路を通じて AI Sonar に接続します。

実際には、これは次のことを意味します:
- アプリが完全なエンドポイント URL を明示的に要求しない限り、ベース URL として `https://api.aisonar.dev/v1` を使用します
- OpenAI スタイルの API キーとして AI Sonar API キーを使用します
- アプリが標準的なチャット運用フローで使用できるモデルを選択します

<Note>
一般的なチャットアプリは通常、**chat-completions 指向**であり、Codex Responses WebSocket クライアントではありません。Codex CLI/App、Cursor、または Responses 固有の動作を必要とするその他のコーディングツールについては、代わりに専用の統合ガイドを使用してください。
</Note>

<Note>
**種類**: チャットアプリ

**主要パス**: OpenAI互換の chat-completions パス

**サポートレベル**: サポート済みパス
</Note>

## 対応アプリケーション

| アプリケーション | プラットフォーム | 備考 |
|-------------|----------|-------|
| Open WebUI | Web、セルフホスト | 人気のセルフホスト型クライアント。OpenAI互換の接続に対応 |
| LibreChat | Web、セルフホスト | カスタム OpenAI エンドポイントに対応したマルチプロバイダーのチャット UI |
| LobeChat | Web、Desktop、セルフホスト | 強力なカスタムプロバイダー対応 |
| Cherry Studio | Windows、macOS、Linux | 優れた OpenAI 互換プロバイダーフロー |
| Chatbox | Windows、macOS、Linux | シンプルな OpenAI互換セットアップ |
| TypingMind | Web、Desktop | カスタム エンドポイント形式を使用 |
| NextChat | Web、セルフホスト | 軽量な ChatGPT 風クライアント。ベース URL の上書きに対応 |
| ChatWise | Windows、macOS | 標準的なカスタムプロバイダーパターン |

## デフォルト接続値

| 項目 | 値 |
|-------|-------|
| ベースURL | `https://api.aisonar.dev/v1` |
| APIキー | `sk-your-api-key` |
| APIタイプ | OpenAI互換 |

アプリがベース URL ではなく**完全なエンドポイント URL**を要求する場合は、次を使用してください:
- Chat Completions: `https://api.aisonar.dev/v1/chat/completions`

## アプリ設定例

<Tabs>
  <Tab title="LobeChat">
    **セルフホスト版 LobeChat**

    ```bash
    OPENAI_API_KEY=sk-your-api-key
    OPENAI_PROXY_URL=https://api.aisonar.dev/v1
    ```

    **LobeChat Cloud**
    1. **Settings** → **AI サービスプロバイダー** を開きます
    2. **OpenAI** を選択します
    3. AI Sonar API キーを入力します
    4. カスタムエンドポイントを `https://api.aisonar.dev/v1` に設定します
  </Tab>

  <Tab title="Cherry Studio">
    1. **Settings** → **モデルプロバイダー** を開きます
    2. **OpenAI 互換** プロバイダーを追加します
    3. 次を入力します:
       - **Name**: AI Sonar
       - **API Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
    4. 使用したい model を追加します
  </Tab>

  <Tab title="Chatbox">
    1. **Settings** → **AI Model** を開きます
    2. **OpenAI API Compatible** を選択します
    3. 次を入力します:
       - **API Host**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
       - **Model**: `gpt-5-mini` または `gpt-4o`
  </Tab>

  <Tab title="TypingMind">
    1. **Settings** を開きます
    2. **カスタム エンドポイント** を選択します
    3. 次を入力します:
       - **エンドポイント URL**: `https://api.aisonar.dev/v1/chat/completions`
       - **API Key**: `sk-your-api-key`
    4. 公開したい model を設定します
  </Tab>

  <Tab title="Open WebUI">
    1. **Admin Settings** → **Connections** を開きます
    2. **OpenAI 互換** 接続を追加します
    3. 次を入力します:
       - **Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
    4. 保存し、公開したい model を選びます
  </Tab>

  <Tab title="LibreChat">
    LibreChat のプロバイダー設定に OpenAI 互換 のエンドポイントを追加し、次を指定します:

    ```yaml
    customEndpoints:
      - name: AI Sonar
        apiKey: '${OPENAI_API_KEY}'
        baseURL: 'https://api.aisonar.dev/v1'
        models:
          default: ['gpt-5-mini', 'gpt-5.4', 'claude-sonnet-4-6']
          fetch: false
    ```
  </Tab>
</Tabs>

## より安全なモデルのデフォルト

| カテゴリ | 推奨モデル |
|----------|------------------|
| 一般的なデフォルト | `gpt-5.4`, `gpt-5-mini`, `gpt-4o` |
| Anthropic | `claude-sonnet-4-6`, `claude-opus-4-6` |
| Google | `gemini-2.5-flash`, `gemini-2.5-pro` |
| DeepSeek | `deepseek-r1`, `deepseek-v3-2` |

## ヒント

<AccordionGroup>
  <Accordion title="アプリが対応している場合はベース URL を優先する">
    ほとんどのアプリでは、完全なエンドポイント URL ではなく、ベース URL として `https://api.aisonar.dev/v1` を指定する方が設定しやすく、壊れにくくなります。
  </Accordion>

  <Accordion title="まずは小さなモデルから使う">
    日常的なチャットには `gpt-5-mini` または `gpt-4o` から始めて、その後、より重い推論タスクには `gpt-5.4` または `claude-opus-4-6` に移行してください。
  </Accordion>

  <Accordion title="コーディングツールは分けて運用する">
    一般的なチャットアプリとコーディングエージェントでは、必要なエンドポイントの動作が異なることがよくあります。Codex/Cursor 系のツールは専用の統合ページで管理してください。
  </Accordion>
</AccordionGroup>

## トラブルシューティング

<AccordionGroup>
  <Accordion title="接続エラー">
    - ベース URL が正確に `https://api.aisonar.dev/v1` であることを確認してください
    - アプリがパスを自動追加する場合は、末尾のスラッシュを削除してください
    - アプリが完全なエンドポイント URL を必要とする場合は、`https://api.aisonar.dev/v1/chat/completions` を使用してください
  </Accordion>

  <Accordion title="401 Unauthorized">
    - API キーを再確認してください
    - key が AI Sonar ダッシュボードで有効になっていることを確認してください
    - 余分な空白や改行が含まれていないことを確認してください
  </Accordion>

  <Accordion title="モデルを利用できません">
    - model 名が正確であることを確認してください
    - `gpt-5-mini`、`gpt-4o`、または `claude-sonnet-4-6` のような広くサポートされているデフォルトを試してください
    - 現在の利用可否を AI Sonar docs または ダッシュボード で確認してください
  </Accordion>
</AccordionGroup>
