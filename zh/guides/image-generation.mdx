AI Sonar 图像 API 涵盖了多个模型系列，具有不同的请求格式。在假设某个字段适用于每个图像模型之前，请使用公共模型合约和特定于端点的 API 参考。

## 选择端点

| 目标 | 端点 | 备注 |
|------|----------|-------|
| 文本到图像 | `POST /v1/images/generations` | 使用 `prompt` 和明确的 `model` |
| 图像到图像 / 参考图像 | `POST /v1/images/generations` | 当模型支持时，使用 `operation: "image-to-image"` 以及 `image_url`、`image_urls` 或 `reference_image_urls` |
| GPT 图像编辑 | `POST /v1/images/edits` | 支持多部分 `image`、JSON `image_url` / `image_urls` 和官方 `images[]` 引用，适用于支持的编辑模型 |
| 旧版变体 | `POST /v1/images/variations` | 对于新工作，优先使用当前生成或编辑路由 |
| 异步状态 | `GET /v1/tasks/{id}` | 当存在时，首先使用 `poll_url` |

## 模型发现

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=image" \\
  -H "Authorization: Bearer sk-your-api-key"
```

图像模型参数差异很大。在发送 `size`、`quality`、`aspect_ratio`、`resolution`、`async`、`background` 或 `output_format` 之前，请阅读所选模型合约。

## 输入和模型特定字段

- `gpt-image-2` 使用 OpenAI 风格的 `size` 和 `quality`；省略 `size` 或 `quality` 以让 AI Sonar 在支持的情况下使用 `auto`。
- Gemini 和 Nano Banana 图像系列通常使用 `aspect_ratio`；仅在所选模型公开时发送 `resolution`。
- Nano Banana 参考图像请求应在 `/v1/images/generations` 上，使用 `operation: "image-to-image"` 和 `image_urls`，而不是在 `/v1/images/edits` 上。
- `/v1/images/generations` 不接受顶层的 `images[]` 或 `file_id`；这些是编辑流程格式。
- 远程图像 URL 必须是公共的 `http`/`https` URL。避免使用私有网络、嵌入凭证、片段和短期签名 URL。

## 创建请求

```bash
curl https://api.aisonar.dev/v1/images/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-image-2",
    "prompt": "一张干净的陶瓷咖啡杯在胡桃木桌上的产品照片",
    "size": "1024x1024",
    "response_format": "url"
  }'
```

## 轮询结果

某些图像路由返回内联 `data[].url` 或 `data[].b64_json`。其他路由返回 `status: "pending"`、`task_id` 和 `poll_url`。如果您请求异步图像生成或编辑，完成的任务结果将返回 URL；当您特别需要 `b64_json` 时，请使用同步请求。

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \\
  -H "Authorization: Bearer sk-your-api-key"
```

## 常见陷阱

- 不要假设每个图像模型都是同步的。
- 不要从异步图像任务请求 `b64_json`；异步结果是以 URL 为导向的。
- 不要向仅支持比例的 Gemini 图像模型发送 `resolution`。
- 不要在 Nano Banana 图像到图像流程中使用 `/v1/images/edits`。
- 对于同步高分辨率图像请求，将 HTTP 客户端超时设置为至少 `120s`。
