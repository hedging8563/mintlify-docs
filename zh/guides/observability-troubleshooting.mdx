## 概述

使用 request_id、billing_transaction_id、task_id 和控制台日志定位生产问题。

使用 AI Sonar 公开 API 和模型契约；不要依赖供应商执行细节。

## 关键规则

- 在向用户返回成功前持久化 `request_id`, `billing_transaction_id`, `task_id`。
- 使用 AI Sonar 公开 API 和模型契约；不要依赖供应商执行细节。
- 让重试保持幂等，避免重复生成或重复计费。

## 生产检查清单

- 在向用户返回成功前持久化 `request_id`, `billing_transaction_id`, `task_id`。
