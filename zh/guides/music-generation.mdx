---
title: "音乐生成"
description: "使用 AI Sonar 音乐任务创建歌曲、器乐、歌词和续集。"
---

音乐生成是异步的。 `POST /v1/music/generations` 创建一个任务并返回 `id` / `task_id`、`status`，通常还会返回 `poll_url`。最终音频和元数据在轮询后出现。

## 模型发现

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \\
  -H "Authorization: Bearer sk-your-api-key"
```

当前公共合同公开了 `suno_music` 用于音乐生成和 `suno_lyrics` 用于仅歌词流。在生产使用之前查询模型目录，而不是固定提供者假设。

## 创建请求

```bash
curl https://api.aisonar.dev/v1/music/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "suno_music",
    "prompt": "一首充满活力的合成流行曲，伴有温暖的声乐和干净的合唱",
    "title": "晨间静电",
    "tags": "合成流行, 充满活力",
    "action": "MUSIC"
  }'
```

对于歌曲或器乐使用 `action: "MUSIC"`。仅在支持仅歌词生成的模型中使用 `action: "LYRICS"`。续集字段如 `continue_clip_id` 和 `continue_at` 用于续集工作流，仅在您的产品继续之前的片段时发送。

## 轮询结果

首先使用返回的 `poll_url`。如果您需要固定端点，请使用 `GET /v1/tasks/{id}`，并使用创建响应中的相同 `id` / `task_id`。

完成的音乐任务可以包括 `audio_url`、`video_url`、`title`、`lyrics` 和模型标准化的元数据。将最终的 URL 存储在您的应用程序中，而不是在终端状态后重复轮询。

## 常见陷阱

- 在任务 `completed` 且存在 `audio_url` 之前，不要显示最终播放器。
- 不要在您的应用状态中使用提供者任务 ID 或提供者获取 URL。
- 不要在一个 UI 后面混合仅歌词和完整音乐操作，而不向用户显示区别。
- 不要在每次页面刷新时重新创建任务；从存储的 `task_id` 恢复。
