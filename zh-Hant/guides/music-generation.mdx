title: "音樂生成"
description: "使用 AI Sonar 音樂任務創建歌曲、器樂、歌詞和延續。"
---

音樂生成是異步的。 `POST /v1/music/generations` 創建一個任務並返回 `id` / `task_id`、`status`，通常還會返回 `poll_url`。最終音頻和元數據在輪詢後出現。

## 模型發現

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \\
  -H "Authorization: Bearer sk-your-api-key"
```

當前公共合約公開了 `suno_music` 用於音樂生成，以及 `suno_lyrics` 用於僅歌詞流程。在生產使用之前查詢模型目錄，而不是固定提供者假設。

## 創建請求

```bash
curl https://api.aisonar.dev/v1/music/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "suno_music",
    "prompt": "一首充滿活力的合成流行曲，帶有溫暖的聲音和清晰的合唱",
    "title": "晨間靜電",
    "tags": "合成流行, 充滿活力",
    "action": "MUSIC"
  }'
```

對於歌曲或器樂，使用 `action: "MUSIC"`。僅在支持僅歌詞生成的模型中使用 `action: "LYRICS"`。延續字段如 `continue_clip_id` 和 `continue_at` 用於延續工作流程，應僅在您的產品正在延續先前片段時發送。

## 輪詢結果

首先使用返回的 `poll_url`。如果您需要固定端點，請使用 `GET /v1/tasks/{id}`，並使用創建響應中的相同 `id` / `task_id`。

完成的音樂任務可以包括 `audio_url`、`video_url`、`title`、`lyrics` 和模型標準化的元數據。將最終的 URL 存儲在您的應用程序中，而不是在終端狀態後重複輪詢。

## 常見陷阱

- 在任務 `completed` 並且存在 `audio_url` 之前，不要顯示最終播放器。
- 不要在您的應用狀態中使用提供者任務 ID 或提供者獲取 URL。
- 不要在一個 UI 中混合僅歌詞和完整音樂操作，而不向用戶顯示區別。
- 不要在每次頁面刷新時重新創建任務；從存儲的 `task_id` 繼續。
