---
title: "影像生成"
description: "建立文本到影像、影像到影像及影像編輯工作流程，針對 AI Sonar 公共影像合約。"
---

AI Sonar 影像 API 涵蓋幾個模型系列，具有不同的請求格式。在假設某個欄位在每個影像模型中都有效之前，請使用公共模型合約和特定端點的 API 參考。

## 選擇端點

| 目標 | 端點 | 備註 |
|------|----------|-------|
| 文本到影像 | `POST /v1/images/generations` | 使用 `prompt` 和明確的 `model` |
| 影像到影像 / 參考影像 | `POST /v1/images/generations` | 當模型支持時，使用 `operation: "image-to-image"` 加上 `image_url`、`image_urls` 或 `reference_image_urls` |
| GPT 影像編輯 | `POST /v1/images/edits` | 支持多部分 `image`、JSON `image_url` / `image_urls`，以及官方 `images[]` 參考，適用於支持的編輯模型 |
| 過時變體 | `POST /v1/images/variations` | 對於新工作，優先使用當前生成或編輯路徑 |
| 非同步狀態 | `GET /v1/tasks/{id}` | 當存在時，首先使用 `poll_url` |

## 模型發現

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=image" \\
  -H "Authorization: Bearer sk-your-api-key"
```

影像模型參數差異很大。在發送 `size`、`quality`、`aspect_ratio`、`resolution`、`async`、`background` 或 `output_format` 之前，請閱讀所選模型合約。

## 輸入和模型特定欄位

- `gpt-image-2` 使用 OpenAI 風格的 `size` 和 `quality`；省略 `size` 或 `quality` 以讓 AI Sonar 在支持的情況下使用 `auto`。
- Gemini 和 Nano Banana 影像系列通常使用 `aspect_ratio`；僅在所選模型公開時發送 `resolution`。
- Nano Banana 參考影像請求應放在 `/v1/images/generations`，並使用 `operation: "image-to-image"` 和 `image_urls`，而不是 `/v1/images/edits`。
- `/v1/images/generations` 不接受頂層的 `images[]` 或 `file_id`；這些是編輯流程格式。
- 遠端影像 URL 必須是公共的 `http`/`https` URL。避免使用私有網絡、嵌入憑證、片段和短期簽名 URL。

## 創建請求

```bash
curl https://api.aisonar.dev/v1/images/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-image-2",
    "prompt": "一張乾淨的陶瓷咖啡杯在胡桃木桌上的產品照片",
    "size": "1024x1024",
    "response_format": "url"
  }'
```

## 輪詢結果

某些影像路徑返回內聯的 `data[].url` 或 `data[].b64_json`。其他路徑返回 `status: "pending"`、`task_id` 和 `poll_url`。如果您請求非同步影像生成或編輯，完成的任務結果將返回 URL；當您特別需要 `b64_json` 時，請使用同步請求。

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \\
  -H "Authorization: Bearer sk-your-api-key"
```

## 常見陷阱

- 不要假設每個影像模型都是同步的。
- 不要從非同步影像任務請求 `b64_json`；非同步結果是以 URL 為導向的。
- 不要對僅比例的 Gemini 影像模型發送 `resolution`。
- 不要對 Nano Banana 影像到影像流程使用 `/v1/images/edits`。
- 對於同步高解析度影像請求，將 HTTP 客戶端超時設置為至少 `120s`。
