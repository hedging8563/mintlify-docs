## 路徑參數

<ParamField path="id" type="string" required>
  從建立影片請求回傳的任務 ID。將 `id` 與 `task_id` 視為相同的非同步識別碼。
</ParamField>

<Info>
如果 create 回應回傳 `poll_url`，請呼叫該 URL。本當它指向 `/v1/tasks/{id}` 時，視為固定的標準狀態端點。

如果該任務不再存在或無法透過公開的非同步任務協定解析，AI Sonar 會回傳 `async_task_not_found` 並附上訊息 `Task not found or no longer available.`
</Info>

## 回應

<ResponseField name="id" type="string">
  正規的非同步任務識別符。
</ResponseField>

<ResponseField name="task_id" type="string">
  非同步任務識別符的別名。
</ResponseField>

<ResponseField name="poll_url" type="string">
  當建立回應提供時，建議用來輪詢的 URL。
</ResponseField>

<ResponseField name="billing_transaction_id" type="string">
  當結算已完成時返回 AI Sonar 帳單交易 ID。它對應 dashboard / 對帳使用的交易識別，與非同步 `id` / `task_id` 不同。
</ResponseField>

<ResponseField name="status" type="string">
  任務狀態：`pending`、`processing`、`completed`、`failed`。
</ResponseField>

<ResponseField name="progress" type="number">
  進度百分比（0-100）。
</ResponseField>

<ResponseField name="video_url" type="string">
  產生影片的 URL（完成時）。
</ResponseField>

<ResponseField name="video" type="object">
  單一影片資料，當可用時包含 `url`、`duration`、`width` 與 `height`。
</ResponseField>

<ResponseField name="videos" type="array">
  當上游回傳多個輸出時的多重影片資料。
</ResponseField>

<ResponseField name="error" type="string">
  錯誤訊息（若失敗）。
</ResponseField>

<ResponseField name="created" type="integer">
  建立時間戳記。
</ResponseField>

<ResponseField name="updated" type="integer">
  最後更新時間戳記。
</ResponseField>

<ResponseField name="model" type="string">
  任務使用的模型。
</ResponseField>

import requests
import time

task_id = "ldtask_aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa"

while True:
    response = requests.get(
        f"https://api.aisonar.dev/v1/tasks/{task_id}",
        headers={"Authorization": "Bearer sk-your-api-key"}
    )

    data = response.json()
    print(f"Status: {data['status']}, Progress: {data.get('progress', 0)}%")

    if data["status"] == "completed":
        print(f"Video URL: {data['video_url']}")
        break
    elif data["status"] == "failed":
        print(f"Error: {data['error']}")
        break

    time.sleep(5)  # Poll every 5 seconds

## 輪詢最佳做法

- 每 5 到 10 秒輪詢一次
- 對於較長的任務實作指數退避
- 設定最大逾時（例如：10 分鐘）
- 優雅地處理 `failed` 狀態

```python
import time

def wait_for_video(task_id, max_wait=600, interval=5):
    """Wait for video with timeout."""
    start = time.time()

    while time.time() - start < max_wait:
        response = requests.get(
            f"https://api.aisonar.dev/v1/tasks/{task_id}",
            headers={"Authorization": "Bearer sk-your-api-key"}
        )
        data = response.json()

        if data["status"] == "completed":
            return data["video_url"]
        elif data["status"] == "failed":
            raise Exception(data.get("error", "Video generation failed"))

        time.sleep(interval)

    raise TimeoutError("Video generation timed out")
```
