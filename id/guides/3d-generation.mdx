Generasi 3D bersifat asinkron. `POST /v1/3d/generations` membuat sebuah tugas; respons status yang selesai mengembalikan URL aset model yang dapat diunduh.

## Penemuan Model

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \\
  -H "Authorization: Bearer sk-your-api-key"
```

Jangan menganggap setiap model 3D mendukung baik teks-ke-3D maupun gambar-ke-3D. Periksa kontrak model yang dipilih sebelum mengirim `image`, `image_url`, `format`, `quality`, `style`, atau `seed`.

## Buat Permintaan

```bash
curl https://api.aisonar.dev/v1/3d/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "Sebuah maskot robot low-poly bergaya dengan topologi yang bersih",
    "format": "glb",
    "quality": "standard"
  }'
```

Teks-ke-3D memerlukan `prompt`. Gambar-ke-3D menggunakan `image` atau `image_url` ketika didukung oleh model. Utamakan URL publik untuk input produksi kecuali gambar sumber bersifat pribadi dan klien Anda sengaja mengunggah data base64.

## Pilihan Output

- `format` dapat meminta pengemasan output seperti `glb`, `fbx`, `obj`, atau `usdz` ketika didukung.
- Nilai `quality` seperti `draft`, `standard`, atau `high` mempengaruhi latensi dan biaya ketika model mengeksposnya.
- `style` dan `seed` tergantung pada model; hilangkan mereka kecuali kontrak yang dipilih mendokumentasikannya.

## Memantau Hasil

Gunakan `poll_url` yang dikembalikan terlebih dahulu. Jika Anda memerlukan endpoint tetap, gunakan `GET /v1/tasks/{id}` dengan `id` / `task_id` yang sama dari respons pembuatan.

Tugas yang selesai mengembalikan `model_url` dan mungkin menyertakan URL spesifik format seperti `glb_url`, `fbx_url`, `obj_url`, atau `usdz_url`. Unduh dan simpan aset dalam produk Anda sendiri jika pengguna memerlukan akses ulang.

## Kesalahan Umum

- Jangan mengharapkan aset 3D dalam respons pembuatan; pantau hingga status terminal.
- Jangan menjanjikan setiap format untuk setiap model.
- Jangan mencoba ulang permintaan pembuatan setelah waktu tunggu klien tanpa memeriksa apakah tugas sudah ada.
- Jangan mengekspos ID tugas penyedia di UI Anda.
