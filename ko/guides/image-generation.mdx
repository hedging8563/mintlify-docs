AI Sonar 이미지 API는 다양한 요청 형식을 가진 여러 모델 패밀리를 포함합니다. 모든 이미지 모델에서 필드가 작동한다고 가정하기 전에 공개 모델 계약 및 엔드포인트별 API 참조를 사용하십시오.

## 엔드포인트 선택

| 목표 | 엔드포인트 | 비고 |
|------|----------|-------|
| 텍스트-투-이미지 | `POST /v1/images/generations` | `prompt` 및 명시적인 `model` 사용 |
| 이미지-투-이미지 / 참조 이미지 | `POST /v1/images/generations` | 모델이 지원할 때 `operation: "image-to-image"`와 `image_url`, `image_urls` 또는 `reference_image_urls` 사용 |
| GPT 이미지 편집 | `POST /v1/images/edits` | 지원되는 편집 모델에 대해 멀티파트 `image`, JSON `image_url` / `image_urls`, 및 공식 `images[]` 참조를 지원 |
| 레거시 변형 | `POST /v1/images/variations` | 새로운 작업을 위해 현재 생성 또는 편집 경로를 선호 |
| 비동기 상태 | `GET /v1/tasks/{id}` | 존재할 경우 먼저 `poll_url` 사용 |

## 모델 탐색

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=image" \\
  -H "Authorization: Bearer sk-your-api-key"
```

이미지 모델 매개변수는 매우 다릅니다. `size`, `quality`, `aspect_ratio`, `resolution`, `async`, `background` 또는 `output_format`을 전송하기 전에 선택한 모델 계약을 읽으십시오.

## 입력 및 모델별 필드

- `gpt-image-2`는 OpenAI 스타일의 `size` 및 `quality`를 사용합니다; AI Sonar이 지원하는 경우 `auto`를 사용하도록 `size` 또는 `quality`를 생략하십시오.
- Gemini 및 Nano Banana 이미지 패밀리는 일반적으로 `aspect_ratio`를 사용합니다; 선택한 모델이 노출할 때만 `resolution`을 전송하십시오.
- Nano Banana 참조 이미지 요청은 `/v1/images/generations`에서 `operation: "image-to-image"` 및 `image_urls`와 함께 있어야 하며, `/v1/images/edits`에서는 사용하지 않아야 합니다.
- `/v1/images/generations`는 최상위 `images[]` 또는 `file_id`를 허용하지 않습니다; 이는 편집 흐름 형식입니다.
- 원격 이미지 URL은 공개 `http`/`https` URL이어야 합니다. 개인 네트워크, 내장 자격 증명, 조각 및 단기 서명된 URL은 피하십시오.

## 요청 생성

```bash
curl https://api.aisonar.dev/v1/images/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-image-2",
    "prompt": "호두나무 책상 위에 있는 세라믹 커피 컵의 깨끗한 제품 사진",
    "size": "1024x1024",
    "response_format": "url"
  }'
```

## 결과 폴링

일부 이미지 경로는 인라인 `data[].url` 또는 `data[].b64_json`을 반환합니다. 다른 경로는 `status: "pending"`, `task_id`, 및 `poll_url`을 반환합니다. 비동기 이미지 생성 또는 편집을 요청하는 경우 완료된 작업 결과는 URL을 반환합니다; 특정하게 `b64_json`이 필요할 때는 동기 요청을 사용하십시오.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \\
  -H "Authorization: Bearer sk-your-api-key"
```

## 일반적인 함정

- 모든 이미지 모델이 동기적이라고 가정하지 마십시오.
- 비동기 이미지 작업에서 `b64_json`을 요청하지 마십시오; 비동기 결과는 URL 중심입니다.
- 비율 전용 Gemini 이미지 모델에 `resolution`을 전송하지 마십시오.
- Nano Banana 이미지-투-이미지 흐름에 대해 `/v1/images/edits`를 사용하지 마십시오.
- 동기 고해상도 이미지 요청에 대해 HTTP 클라이언트 타임아웃을 최소 `120s`로 설정하십시오.
