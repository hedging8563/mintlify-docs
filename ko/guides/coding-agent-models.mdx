## AI Sonar 라우팅 작동 방식

### 라우팅 전략

| 전략 | 동작 | 최적 용도 |
|------|------|---------|
| **PRIORITY** | 최고 우선순위 사용 가능 프로바이더로 라우팅 | 안정성 중시 |
| **COST** | 최저가 사용 가능 프로바이더로 라우팅 | 비용 중시 |

### 자동 폴백

```
요청: claude-sonnet-4-6
  → Provider A (기본): 503 에러
  → Provider B (폴백): ✓ 성공
```

### 네이티브 포맷 라우팅

선택한 모델과 라우팅된 채널이 지원하는 경우, AI Sonar는 해당 모델 패밀리의 네이티브 API 형식으로 요청을 라우팅해 호환성을 높일 수 있습니다:

| 모델 패밀리 | API Format | Base URL |
|-------------|------------|----------|
| Claude | Anthropic Messages | `https://api.aisonar.dev` |
| GPT | OpenAI Responses | `https://api.aisonar.dev/v1` |
| Gemini | Gemini Native API | 클라이언트 경로에 따라 `https://api.aisonar.dev` 또는 `https://api.aisonar.dev/v1beta` |
| DeepSeek | OpenAI Chat | `https://api.aisonar.dev/v1` |

Claude Code는 강한 Anthropic-native 경로입니다. Gemini-native API도 존재하지만, Gemini CLI 자체는 여전히 best-effort 호환 경로일 뿐 장기적으로 안정적인 워크플로로 간주하면 안 됩니다.

    ```bash
    export ANTHROPIC_BASE_URL="https://api.aisonar.dev"
    ```

  </Accordion>
