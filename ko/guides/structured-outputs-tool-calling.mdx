구조화된 출력 및 도구는 실제 AI Sonar API 경로이지만, 모든 모델과 경로에서 보편적인 기능은 아닙니다. 필요한 동작을 소유한 경로를 선택하세요.

## JSON 및 구조화된 출력

이식 가능한 구조화된 응답을 위해 Chat Completions `response_format: {"type":"json_object"}`로 시작하고 애플리케이션에서 출력을 검증하세요. 이는 AI Sonar 공유 검증에서 안정적인 공개 요청 형식입니다.

```json
{
  "model": "gpt-5.4",
  "messages": [{"role": "user", "content": "Return a JSON object with city and weather."}],
  "response_format": {"type": "json_object"}
}
```

`json_schema` 및 `strict` 구조화된 출력 경로는 일부 업스트림 및 변환 코드에 존재하지만, 이는 모델 및 경로에 따라 다릅니다. 이를 선택한 모델로 검증할 수 있는 기능으로 취급하고, 모든 AI Sonar 경로에 걸쳐 보편적인 약속으로 간주하지 마세요.

## 도구 호출 경계

| 도구 형태 | 선호할 경로 | 생산 노트 |
|------------|-----------------|-----------------|
| OpenAI 함수 도구 | `/v1/chat/completions` | 사용자 정의 함수 호출에 이식 가능 |
| OpenAI 응답 도구 | `/v1/responses` | 응답 전용 내장 또는 호스팅 도구에 사용 |
| Anthropic 도구 | `/v1/messages` | Claude-native 도구 사용 및 사고 흐름에 사용 |
| Gemini 함수 선언 / 내장 도구 | `/v1beta/models/:model:generateContent` | Gemini 경로에서 Gemini-native 도구 필드를 유지 |

AI Sonar은 귀하의 함수를 대신 실행하지 않습니다. 모델이 `tool_calls`, `function_call`, `tool_use` 또는 Gemini 함수 호출 부분을 반환할 때, 귀하의 애플리케이션은 도구를 실행하고, 경로에서 요구하는 형식으로 도구 결과를 추가하며 대화를 계속해야 합니다.

## 일반적인 함정

- 제공자 고유의 내장 도구를 Chat Completions를 통해 전송하고 조용히 변환되기를 기대하지 마세요.
- 선택한 경로와 모델이 테스트되지 않았다면 엄격한 JSON 스키마에 의존하지 마세요.
- 스키마를 작고 명시적으로 유지하세요; 큰 스키마는 토큰을 추가하고 신뢰성을 감소시킬 수 있습니다.
- JSON 모드가 활성화되어 있어도 모델 출력을 서버 측에서 검증하세요.
- 지원되지 않는 필드 오류를 계약 피드백으로 취급하세요. 필드를 제거하거나 이를 소유한 기본 경로를 사용하세요.

## 예시

```bash
curl https://api.aisonar.dev/v1/chat/completions \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-5.4",
    "messages": [{"role":"user","content":"Extract name and email as JSON."}],
    "response_format": {"type":"json_object"},
    "tools": [{
      "type": "function",
      "function": {
        "name": "lookup_customer",
        "description": "Look up a customer by email",
        "parameters": {
          "type": "object",
          "properties": {"email": {"type": "string"}},
          "required": ["email"]
        }
      }
    }]
  }'
```
