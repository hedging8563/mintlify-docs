이 페이지는 AI Sonar의 공개 비동기 작업 계약을 반영합니다. 공급자 작업 URL, 라우팅 ID 또는 상위 콜백 동작을 사용자 대면 통합 계약으로 취급하지 않습니다.

## 공개 계약

AI Sonar은 생성된 미디어에 대해 하나의 공개 비동기 작업 계약을 제공합니다. 생성 엔드포인트는 `id`, `task_id`, `status`, 및 `poll_url`을 반환할 수 있습니다. `id`와 `task_id`를 동일한 공개 작업 ID로 취급하고, 작업 ID와 `poll_url`을 자신의 작업 기록에 저장하십시오.

반환된 `poll_url`을 먼저 사용하십시오. 고정 엔드포인트가 필요한 경우, 생성 응답에서 동일한 `id` / `task_id`를 사용하여 `GET /v1/tasks/{id}`를 사용하십시오.

현재 공개 생성 작업은 폴링 기반입니다. AI Sonar은 내부적으로 상위 콜백 수신기를 가지고 있지만, 이는 고객 웹훅이 아니며 통합 계약으로 사용해서는 안 됩니다.

## 권장 흐름

1. 명시적인 `model`과 함께 생성 요청을 전송합니다.
2. 응답이 인라인인 경우, 최종 결과를 즉시 읽습니다.
3. 응답에 `status: "pending"`, `task_id`, 또는 `poll_url`이 포함된 경우, 작업을 지속하고 폴링을 시작합니다.
4. 긴 미디어 작업의 경우 매 `5-10초`마다 폴링합니다. 최종 상태에서만 중지합니다.
5. `completed` 상태에서 해당 미디어 유형의 결과 필드를 읽습니다. `failed` 상태에서는 공개 오류를 저장하고, 재시도가 새로운 사용자 가시 작업을 생성할 때만 재시도 작업을 표시합니다.

```json
{
  "id": "ldtask_aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa",
  "task_id": "ldtask_aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa",
  "status": "pending",
  "poll_url": "/v1/tasks/ldtask_aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa",
  "model": "veo3.1"
}
```

## 폴링 결과

```bash
curl "https://api.aisonar.dev/v1/tasks/ldtask_aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa" \\
  -H "Authorization: Bearer sk-your-api-key"
```

예상되는 공개 상태는 `pending`, `processing`, `completed`, 및 `failed`입니다. 취소된 작업은 `failed`로 표시되며 `cancelled: true` 및 `cancellation_status: "cancelled"`로 나타나므로 이전 상태 처리가 계속 작동합니다.

공급자 특정 URL, 상위 작업 ID, 라우팅 ID 또는 개인 실행 메타데이터를 폴링하지 마십시오. 이들은 안정적인 공개 계약이 아닙니다.

## 청구 및 조정

- 생성 응답은 비동기 작업에 대한 예상 금액을 예약할 수 있습니다.
- 최종 정산은 작업이 최종 상태에 도달한 후에 발생합니다.
- 정산이 가능할 때, 작업 상태 응답은 `billing_transaction_id` 및 `X-Billing-Transaction-ID` 헤더를 노출할 수 있습니다.
- 작업 생성 후 클라이언트 타임아웃이 발생하면, 작업을 맹목적으로 재생성하지 마십시오. 저장된 `task_id` / `poll_url`에서 재개하십시오.

## 취소

`DELETE /v1/tasks/{id}`는 의도적으로 좁습니다. 현재 `seedance-1.5-pro`, `seedance-2.0`, 및 `seedance-2.0-fast`와 같은 대기 중인 Volcengine Seedance 비디오 작업을 지원합니다. 지원되지 않는 작업은 `400 unsupported_task_cancel`을 반환하며, 이미 실행 중이거나 최종 상태인 작업은 `409 task_not_cancellable`을 반환합니다.

## 문제 해결 체크리스트

- `async_task_not_found`: 작업이 알려지지 않았거나, 만료되었거나, 이 API 키에 접근할 수 없거나, 공개 비동기 작업 계약의 일부가 아닙니다.
- 최종 미디어 URL 누락: 최종 상태에 도달할 때까지 폴링을 계속하십시오; `pending` 응답이 출력을 포함한다고 가정하지 마십시오.
- 중복 작업: 재시도 경로가 HTTP 타임아웃 후 두 번째 작업을 생성했는지 확인하십시오.
- 청구 불일치: 대시보드 또는 관리 API에서 `request_id`, `task_id`, 및 `billing_transaction_id`를 비교하십시오.

지원에 연락할 때, `request_id`, `task_id`, `billing_transaction_id`가 있을 경우 포함하고, 엔드포인트, 모델, 타임스탬프, 및 정리된 요청 형식을 포함하십시오. 지원에서 수정된 샘플을 요청하지 않는 한 API 키, 개인 미디어 또는 전체 프롬프트를 포함하지 마십시오.
