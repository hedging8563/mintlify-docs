description: "OpenAI, Anthropic, Gemini 및 미디어 워크로드를 AI Sonar으로 최소한의 안전한 변경으로 이동합니다."
---

AI Sonar은 네이티브 우선이며 OpenAI와 호환됩니다. 모든 워크로드를 하나의 형식으로 변환하는 대신 필요한 클라이언트 동작에 맞는 경로를 선택하세요.

## 형식 선택

| 기존 워크로드 | AI Sonar 기본 URL | 주요 엔드포인트 | 비고 |
|-------------------|-------------------|------------------|-------|
| OpenAI 채팅 완성 | `https://api.aisonar.dev/v1` | `/chat/completions` | OpenAI 호환 클라이언트에 대한 최소한의 변경 |
| OpenAI 응답 | `https://api.aisonar.dev/v1` | `/responses` | 응답 전용 동작에만 사용 |
| Anthropic SDK | `https://api.aisonar.dev` | `/v1/messages` | SDK 기본 URL에 `/v1`을 추가하지 마세요 |
| Gemini REST | `https://api.aisonar.dev` | `/v1beta/models/:model:generateContent` | 네이티브 경로에서 Gemini 네이티브 필드를 유지하세요 |
| 미디어 생성 | `https://api.aisonar.dev/v1` | `/images`, `/videos`, `/music`, `/3d` | `recommended_for`로 모델을 발견하고 문서화된 곳에서 비동기 폴링을 기대하세요 |

## OpenAI 호환 마이그레이션

```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.aisonar.dev/v1",
)

response = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Hello from AI Sonar"}],
)
```

기존의 재시도, 타임아웃 및 스트리밍 코드를 유지하되, 모델 ID를 `GET /v1/models`에 대해 검증하세요. 이미지 생성을 위해서는 `model`을 명시적으로 전송하고 이미지 모델이 채팅 모델보다 다르므로 이미지 가이드를 읽으세요.

## Anthropic 마이그레이션

```python
from anthropic import Anthropic

client = Anthropic(
    api_key="sk-your-api-key",
    base_url="https://api.aisonar.dev",
)

message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Explain AI Sonar in one sentence."}],
)
```

메시지 도구 사용 및 확장된 사고와 같은 Claude 네이티브 동작을 위해 이 경로를 사용하세요. 기능이 Anthropic 네이티브인 경우, Chat Completions를 통해 변환하는 대신 `/v1/messages`에 유지하세요.

## Gemini 마이그레이션

```bash
curl "https://api.aisonar.dev/v1beta/models/gemini-3.5-flash:generateContent" \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{"contents":[{"parts":[{"text":"Hello"}]}]}'
```

Gemini 내장 도구, 파일 API 리소스, 캐시된 콘텐츠 및 함수 선언은 Gemini 네이티브 동작이 필요할 때 `/v1beta`에 유지해야 합니다.

## 미디어 마이그레이션

1. 모델을 선택하기 전에 `GET /v1/models?recommended_for=image|video|music|3d`를 쿼리하세요.
2. `aisonar.public_contract_summary`를 읽고, 상세 페이지에서 전체 `aisonar.public_contract`를 확인하세요.
3. 명시적인 `model`을 전송하세요. 프로덕션 트래픽에 대해 레거시 기본값에 의존하지 마세요.
4. 비동기 작업을 위해 `task_id`와 `poll_url`을 저장하세요.
5. 제공자 작업 ID가 아닌 사용 기록을 통해 비용을 조정하세요.

## 프로덕션 롤아웃

- 하나의 엔드포인트와 하나의 모델 패밀리로 시작하세요.
- 동기 이미지 경로에 대해 HTTP 타임아웃을 충분히 높게 설정하세요; 긴 작업에 대해 사용 가능한 경우 비동기 모드를 사용하세요.
- 프로덕션 트래픽을 전송하기 전에 공용 식별자와 정제된 요청 형식을 기록하세요.
- 응답 샘플, 사용 기록 및 고객 가시 출력물을 비교하세요.
- 롤백을 간단하게 유지하세요: AI Sonar 트래픽이 안정될 때까지 이전 제공자 클라이언트를 보존하세요.
