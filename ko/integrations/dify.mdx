## 개요

Dify는 OpenAI 호환 모델 제공자 흐름을 통해 AI Sonar와 잘 작동합니다.

현재 Dify 버전에서는 일반적으로 가장 안전한 방법은 다음과 같습니다:
- 기본 제공 **OpenAI** 제공자 선택
- AI Sonar API key 설정
- `https://api.aisonar.dev/v1`의 사용자 지정 base URL 설정

<Note>
일부 구형 Dify 빌드에서는 사용자 지정 base URL 필드가 있는 기본 제공 `OpenAI` 제공자 대신 이를 `OpenAI-API-compatible`로 노출합니다. Dify UI가 다르게 보인다면, 해당 버전에서 사용 가능한 가장 가까운 OpenAI 호환 사용자 지정 제공자 흐름을 사용하세요.
</Note>

<Note>
**유형**: 프레임워크 / 플랫폼

**주요 경로**: OpenAI 호환 채팅 경로

**지원 수준**: 범위 제한 지원
</Note>

이 경로는 chat-completions 지향 통합 경로입니다. Dify가 전용 Codex 통합과 동일한 Responses 또는 WebSocket 동작을 노출한다고 보장하지 않습니다.

## 사전 요구 사항

- API access가 가능한 AI Sonar 계정
- Dify Cloud 또는 자체 호스팅 Dify

## 구성 단계

### 1단계: API Key 가져오기

1. [AI Sonar Dashboard](https://console.aisonar.dev)에 로그인합니다
2. [API Keys](https://console.aisonar.dev/api)를 엽니다
3. `sk-`로 시작하는 API key를 생성하거나 복사합니다

### 2단계: 제공자 구성

<Steps>
  <Step title="Model 제공자s 열기">
    Dify에서 **Settings** → **Model 제공자s**로 이동합니다
  </Step>
  <Step title="OpenAI 선택">
    **OpenAI** 제공자 설정을 엽니다. 사용 중인 Dify 버전에서 여기에서 사용자 지정 base URL을 제공하지 않는 경우, 대신 해당 버전에서 노출하는 OpenAI 호환 사용자 지정 제공자 옵션을 사용하세요.
  </Step>
  <Step title="AI Sonar 설정 입력">
    다음 값을 사용합니다:

    | 필드 | 값 |
    |-------|-------|
    | API 키 | `sk-your-api-key` |
    | API 기본 URL / 사용자 지정 기본 URL | `https://api.aisonar.dev/v1` |
  </Step>
  <Step title="Model 추가">
    사용하려는 model을 추가합니다. 예를 들면 다음과 같습니다:
    - `gpt-5.4`
    - `gpt-5-mini`
    - `gpt-4o`
    - `claude-sonnet-4-6`
    - `claude-opus-4-6`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### 3단계: 연결 테스트

1. `gpt-5-mini` 또는 `gpt-4o`와 같은 model 하나를 선택합니다
2. 테스트 prompt를 전송합니다
3. Dify가 유효한 응답을 받는지 확인합니다

## Knowledge Base용 Embeddings

RAG 및 knowledge base indexing의 경우, 다음과 같은 embedding model을 추가합니다:
- `text-embedding-3-small`
- `text-embedding-3-large`

그런 다음 관련 knowledge base 또는 application 설정에서 이를 기본 embedding model로 설정합니다.

## 권장 Model 역할

| 사용 사례 | 권장 모델 |
|----------|------------------|
| 기본 chat | `gpt-5.4`, `gpt-5-mini`, `gpt-4o` |
| 심층 추론 | `gpt-5.4`, `claude-opus-4-6`, `deepseek-r1` |
| 빠름/저렴함 | `gpt-5-mini`, `gemini-2.5-flash` |
| Embeddings | `text-embedding-3-small`, `text-embedding-3-large` |

## 문제 해결

<AccordionGroup>
  <Accordion title="연결 오류">
    - base URL이 정확히 `https://api.aisonar.dev/v1`인지 확인합니다
    - Dify가 후행 슬래시를 중복 추가하는 경우 이를 제거합니다
    - Dify server가 public internet을 통해 AI Sonar에 도달할 수 있는지 확인합니다
  </Accordion>

  <Accordion title="401 Unauthorized">
    - API key를 다시 확인합니다
    - AI Sonar dashboard에서 key가 활성 상태인지 확인합니다
    - Dify에 붙여넣은 값에 불필요한 공백이나 줄바꿈이 포함되지 않았는지 확인합니다
  </Accordion>

  <Accordion title="Model을 찾을 수 없음">
    - model 이름이 정확한지 확인합니다
    - 제공자 UI가 이전 값을 캐시한 경우 모델 항목을 다시 추가합니다
    - AI Sonar docs 또는 dashboard에서 현재 사용 가능한 model을 확인합니다
  </Accordion>
</AccordionGroup>
