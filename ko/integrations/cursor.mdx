## 개요

<Note>
**유형**: 코딩 도구

**주요 경로**: OpenAI 호환

**지원 수준**: 제한적으로 지원
</Note>

[Cursor](https://cursor.sh)는 BYOK(Bring-Your-Own-Key) 방식의 제공자를 지원하지만, 커스텀 API 키는 표준 채팅 모델에서만 작동합니다. 여기서는 AI Sonar를 Cursor의 모든 모델 기능을 완전히 대체하는 수단이 아니라, 채팅형 에디터 작업을 위한 OpenAI 호환 엔드포인트로 사용하세요.

## 구성

1. Cursor Settings를 엽니다 (`Cmd/Ctrl + ,`)
2. 왼쪽 사이드바에서 **Models**로 이동합니다
3. 아래로 스크롤하여 **OpenAI API Key** 섹션으로 이동합니다
4. AI Sonar API 키를 입력합니다
5. **Override OpenAI Base URL**을 활성화합니다
6. Base URL을 다음과 같이 설정합니다:

```
https://api.aisonar.dev/v1
```

<Warning>
Cursor 커스텀 API 키는 표준 채팅 모델에만 적용됩니다. Tab Completion 및 기타 Cursor가 관리하는 특수 기능은 계속해서 Cursor의 내장 제공자를 사용합니다.
</Warning>

<Warning>
Cursor의 커스텀 API 키 경로는 표준 채팅 모델에만 적용됩니다. Tab Completion 및 일부 추론 전용 흐름처럼 Cursor가 관리하는 특수 모델이 필요한 기능은 계속해서 Cursor의 내장 제공자를 사용합니다. 또한 Cursor는 최종 프롬프트 구성을 위해 요청이 여전히 자체 백엔드를 통과한다고 밝히고 있습니다.
</Warning>

## 모델 선택

구성 후에는 AI Sonar를 통해 제공되는 표준 채팅 모델을 우선적으로 사용하세요:

### 코딩용 권장 모델

| 모델 | 가장 적합한 용도 |
|-------|----------|
| `gpt-5.4` | 일반 코딩, 디버깅, 채팅 |
| `gpt-5-mini` | 빠른 수정 및 가벼운 프롬프트 |

## 가장 잘 작동하는 항목

### 채팅 및 에디터 작업 흐름

Cursor에서 AI Sonar를 다음 용도로 사용하세요:

- 코드 설명
- 버그 수정
- 리팩토링 제안
- 문서 생성

### Cursor 관리 모델에 남는 항목

다음 기능은 안정적으로 AI Sonar 엔드포인트로 전환되지 않습니다:

- 탭 자동 완성
- 특수한 비표준 채팅 모델
- Cursor가 내장 모델에서만 활성화하는 제공자별 기능

## 문제 해결

<AccordionGroup>
  <Accordion title="모델이 표시되지 않음">
    Cursor는 모델 및 제공자 상태를 캐시합니다. API 키와 base URL을 구성한 후 Cursor를 다시 시작하세요.
  </Accordion>

  <Accordion title="BYOK 모드에서 지원되지 않는 모델">
    커스텀 API 키는 표준 채팅 모델에서만 작동합니다. 해당 모델이 특수한 Cursor 인프라를 사용하는 경우 `gpt-5.4`와 같은 표준 채팅 모델로 전환하세요.
  </Accordion>

  <Accordion title="인증 오류">
    API 키가 올바르고 충분한 잔액이 있는지 확인하세요.
  </Accordion>

  <Accordion title="AI Sonar를 활성화한 후 다른 제공자가 작동을 멈춤">
    Cursor의 BYOK 구성은 다른 제공자 경로에 영향을 줄 수 있습니다. OpenAI base URL override를 활성화한 후 다른 모델 경로가 작동하지 않으면 override를 비활성화하고 Cursor를 다시 시작하세요.
  </Accordion>
</AccordionGroup>
