<Note>
Este skill ensina seu agente de programação a integrar as **centenas de APIs de IA** da AI Sonar apenas descrevendo o que você precisa. Ele é empacotado como um skill compartilhado agent-first e funciona com qualquer agente de programação que suporte arquivos de skill ou regras.

A API é **[Agent-First](/guides/agent-first-api)** — mesmo que o agente adivinhe errado o nome de um modelo, a resposta de erro informa exatamente como corrigir isso.
</Note>

<Note>
Esta página documenta uma camada reutilizável de empacotamento de skill, e não uma superfície de compatibilidade de protocolo. Se você precisa de configuração de ponto de terminação, SDK ou cliente, use as páginas de integração dedicadas dessas ferramentas.
</Note>

## Como Funciona

1. **Você descreve o que precisa** — "Quero usar GPT-4 no meu projeto Python"
2. **O agente chama a API** — ele pode adivinhar um nome de modelo ou ponto de terminação
3. **Se algo der errado** — a API retorna dicas estruturadas (`did_you_mean`, `suggestions`, `retry_after`)
4. **O agente se autocorrige** — lê as dicas e tenta novamente com os parâmetros corretos
5. **Você recebe código funcional** — normalmente na primeira ou segunda tentativa

## Instalação

<Note>
Use o comando de instalação canônico não interativo:

```bash
npx skills add https://github.com/hedging8563/aisonar-skills --skill aisonar-api-integration -y
```

Isso instala o skill compartilhado `aisonar-api-integration` do repositório de skills da AI Sonar.

Se sua ferramenta não suportar o instalador, copie a pasta `skills/aisonar-api-integration/` do repositório para o diretório compartilhado de skills ou regras da sua ferramenta.
</Note>

### Verificar Instalação

Pergunte ao seu agente de programação:
```
Quais skills estão disponíveis?
```

Se você vir `aisonar-api-integration`, a instalação foi bem-sucedida.

## Obtenha sua API Key

<Steps>
  <Step title="Visite a AI Sonar">

  </Step>
  <Step title="Faça login">
    Crie uma conta ou entre
  </Step>
  <Step title="Obtenha a API Key">

  </Step>
  <Step title="Copie a chave">
    Sua chave começa com `sk-...` — salve-a com segurança

</Tip>

## Exemplos

### Conclusão de Chat

**Você diz:**
```
Quero usar GPT-4 no meu projeto Python
```

**O agente gera:**
```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["TOKENLAB_API_KEY"],
    base_url="https://api.aisonar.dev/v1"
)

response = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```

Execute com:

```bash
export TOKENLAB_API_KEY="sk-your-api-key"
python app.py
```

Se o agente adivinhar um nome de modelo errado (ex.: `gpt5`), a API retorna:
```json
{
  "error": {
    "code": "model_not_found",
    "did_you_mean": "gpt-5.4",
    "suggestions": [{"id": "gpt-5.4"}, {"id": "gpt-5-mini"}],
    "hint": "Did you mean 'gpt-5.4'? Use GET https://api.aisonar.dev/v1/models to list all available models."
  }
}
```

O agente lê `did_you_mean`, corrige o nome do modelo e tenta novamente — tudo automaticamente.

## Recuperação de Erros Agent-First

Toda resposta de erro inclui dicas estruturadas que qualquer agente de programação pode interpretar. Veja a [referência completa da API Agent-First](/guides/agent-first-api) para mais detalhes.

| Error | O que a API retorna | O que o agente faz |
|-------|---------------------|---------------------|
| Nome de modelo incorreto | `did_you_mean` + `suggestions` | Corrige automaticamente e tenta novamente |
| Saldo insuficiente | `balance_usd` + `suggestions` mais baratas | Muda para um modelo acessível |
| Modelo indisponível | `alternatives` + `retry_after` | Muda para um modelo disponível |
| Rate limited | `retry_after` (segundos exatos) | Aguarda e tenta novamente |
| Contexto muito longo | `suggestions` com modelos maiores | Muda para um modelo com contexto maior |
| Incompatibilidade de contrato em solicitação não chat | `supported_operations`, `supported_parameters`, `request_endpoint`, `recommended_request` | Reconstrói a solicitação a partir do contrato público do modelo |

### Otimização de Endpoint Nativo

Quando você chama `/v1/chat/completions` com um modelo Claude ou Gemini, a API retorna cabeçalhos de otimização:



Trate esses headers como dicas de roteamento, não como uma troca automática obrigatória. Chamadas nativas do Claude usam `/v1/messages`; chamadas nativas do Gemini devem seguir o endpoint indicado por the native endpoint hint header, geralmente `/v1beta/gemini`.

### Descoberta de Modelos

Não é necessário procurar na documentação:

```bash
# Visão geral da API legível por máquina
curl https://api.aisonar.dev/llms.txt

# List models by category
curl "https://api.aisonar.dev/v1/models?category=chat" -H "Authorization: Bearer sk-KEY"
curl "https://api.aisonar.dev/v1/models?category=image" -H "Authorization: Bearer sk-KEY"

# Lista recomendada para cenários não chat
curl "https://api.aisonar.dev/v1/models?recommended_for=image" -H "Authorization: Bearer sk-KEY"
curl "https://api.aisonar.dev/v1/models?recommended_for=translation" -H "Authorization: Bearer sk-KEY"

# Leia o contrato público de um modelo antes de repetir uma solicitação não chat
curl "https://api.aisonar.dev/v1/models/gpt-image-2" -H "Authorization: Bearer sk-KEY"

# Filtrar por capacidade
curl "https://api.aisonar.dev/v1/models?tag=coding" -H "Authorization: Bearer sk-KEY"
```

## FAQ

<AccordionGroup>
  <Accordion title="O skill não está sendo acionado automaticamente?">
    Tente mencionar "AI Sonar" ou "AI Sonar API" na sua solicitação:
    ```
    Quero integrar GPT-4 ao meu projeto usando AI Sonar
    ```
  </Accordion>

  <Accordion title="Como a API ajuda o agente a encontrar o modelo certo?">
    A [API Agent-First](/guides/agent-first-api) da AI Sonar retorna dicas de erro estruturadas. Se o agente adivinhar um nome de modelo errado, o erro inclui `did_you_mean` com o nome correto e `suggestions` com alternativas. O agente lê isso e corrige automaticamente na próxima tentativa.
  </Accordion>

  <Accordion title="Quais agentes de programação são suportados?">
    Qualquer agente de programação que suporte diretórios compartilhados de skills ou regras. Quando o instalador é suportado, `npx skills add` coloca automaticamente o skill compartilhado `aisonar-api-integration` no local detectado.
  </Accordion>

  <Accordion title="Como atualizar o skill?">
    Execute o instalador novamente:
    ```bash
    npx skills add https://github.com/hedging8563/aisonar-skills --skill aisonar-api-integration -y
    ```
  </Accordion>
</AccordionGroup>
