## Visão geral



<Note>
**Tipo**: Ferramenta de código

**Caminho principal**: compatível com OpenAI

**Nível de suporte**: Compatível com limites
</Note>

O [Cursor](https://cursor.sh) oferece suporte a provedores bring-your-own-key (BYOK), mas chaves de API personalizadas funcionam apenas com modelos de chat padrão. Use o AI Sonar aqui como um ponto de terminação compatível com OpenAI para fluxos de trabalho do editor no estilo chat, e não como uma substituição completa para todos os recursos de modelo do Cursor.

## Configuração

1. Abra as Configurações do Cursor (`Cmd/Ctrl + ,`)
2. Navegue até **Models** na barra lateral esquerda
3. Role para baixo até a seção **OpenAI API Key**
4. Insira sua chave de API do AI Sonar
5. Ative **Override OpenAI Base URL**
6. Defina a Base URL como:

```
https://api.aisonar.dev/v1
```

<Warning>
As chaves de API personalizadas do Cursor se aplicam apenas a modelos de chat padrão. O Tab Completion e outros recursos especializados gerenciados pelo Cursor continuarão usando os provedores integrados do Cursor.
</Warning>

<Warning>
O fluxo de chave de API personalizada do Cursor se aplica apenas a modelos de chat padrão. Recursos que exigem modelos especializados gerenciados pelo Cursor, como Tab Completion e alguns fluxos específicos de reasoning, continuarão usando os provedores integrados do Cursor. O Cursor também observa que as solicitações ainda passam pelo seu servidor intermediário para a montagem final do prompt.
</Warning>

## Seleção de modelo

Após a configuração, prefira modelos de chat padrão expostos por meio do AI Sonar:

### Modelos recomendados para programação

| Modelo | Melhor para |
|-------|----------|
| `gpt-5.4` | Programação geral, debugging, chat |
| `gpt-5-mini` | Edições rápidas e prompts leves |

## O que funciona melhor

### Fluxos de chat e editor

Use o AI Sonar por meio do Cursor para:

- Explicações de código
- Correção de bugs
- Sugestões de refatoração
- Geração de documentação

### O que permanece em modelos gerenciados pelo Cursor

Estes recursos não mudam de forma confiável para o seu ponto de terminação do AI Sonar:

- Conclusão por Tab
- Modelos especializados que não são de chat padrão
- Recursos específicos de provedor que o Cursor só habilita em modelos integrados

## Solução de problemas

<AccordionGroup>
  <Accordion title="Modelos não aparecem">
    O Cursor armazena em cache o estado do modelo e do fornecedor. Reinicie o Cursor após configurar a chave de API e a URL de base.
  </Accordion>

  <Accordion title="Modelo não suportado no modo BYOK">
    Chaves de API personalizadas funcionam apenas com modelos de chat padrão. Se um modelo usar infraestrutura especializada do Cursor, mude para um modelo de chat padrão, como `gpt-5.4`.
  </Accordion>

  <Accordion title="Erros de autenticação">
    Verifique se sua chave de API está correta e tem saldo suficiente.
  </Accordion>

  <Accordion title="Outro provedor parou de funcionar após ativar o AI Sonar">
    A configuração BYOK do Cursor pode afetar outros fluxos de fornecedor. Se outro caminho de modelo parar de funcionar após ativar a substituição da URL de base OpenAI, desative a substituição e reinicie o Cursor.
  </Accordion>
</AccordionGroup>
