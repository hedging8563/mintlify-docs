---
title: "Geração de Imagem"
description: "Construa fluxos de trabalho de texto-para-imagem, imagem-para-imagem e edição de imagem contra contratos públicos de imagem do AI Sonar."
---

As APIs de imagem do AI Sonar cobrem várias famílias de modelos com diferentes formatos de solicitação. Use o contrato de modelo público e a referência da API específica do endpoint antes de assumir que um campo funciona em todos os modelos de imagem.

## Escolha O Endpoint

| Objetivo | Endpoint | Notas |
|----------|----------|-------|
| Texto-para-imagem | `POST /v1/images/generations` | Use `prompt` e um `model` explícito |
| Imagem-para-imagem / imagem de referência | `POST /v1/images/generations` | Use `operation: "image-to-image"` mais `image_url`, `image_urls` ou `reference_image_urls` quando o modelo suportar |
| Edição de imagem GPT | `POST /v1/images/edits` | Suporta `image` multipart, `image_url` / `image_urls` em JSON, e referências oficiais `images[]` para modelos de edição suportados |
| Variação legada | `POST /v1/images/variations` | Prefira rotas de geração ou edição atuais para novos trabalhos |
| Status assíncrono | `GET /v1/tasks/{id}` | Use `poll_url` primeiro quando presente |

## Descoberta de Modelos

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=image" \\
  -H "Authorization: Bearer sk-your-api-key"
```

Os parâmetros do modelo de imagem diferem bastante. Leia o contrato do modelo selecionado antes de enviar `size`, `quality`, `aspect_ratio`, `resolution`, `async`, `background` ou `output_format`.

## Entradas E Campos Específicos do Modelo

- `gpt-image-2` usa `size` e `quality` no estilo OpenAI; omita `size` ou `quality` para permitir que o AI Sonar use `auto` onde suportado.
- As famílias de imagem Gemini e Nano Banana geralmente usam `aspect_ratio`; envie `resolution` apenas quando o modelo selecionado expô-lo.
- Solicitações de imagem de referência do Nano Banana pertencem a `/v1/images/generations` com `operation: "image-to-image"` e `image_urls`, não em `/v1/images/edits`.
- `/v1/images/generations` não aceita `images[]` ou `file_id` de nível superior; esses são formatos de fluxo de edição.
- URLs de imagem remotas devem ser URLs públicas `http`/`https`. Evite redes privadas, credenciais incorporadas, fragmentos e URLs assinadas de curta duração.

## Criar Solicitação

```bash
curl https://api.aisonar.dev/v1/images/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-image-2",
    "prompt": "Uma foto de produto limpa de uma xícara de café de cerâmica em uma mesa de nogueira",
    "size": "1024x1024",
    "response_format": "url"
  }'
```

## Consultando Resultados

Algumas rotas de imagem retornam `data[].url` ou `data[].b64_json` inline. Outras rotas retornam `status: "pending"`, `task_id` e `poll_url`. Se você solicitar geração ou edição de imagem assíncrona, os resultados da tarefa concluída retornam URLs; use solicitações síncronas quando precisar especificamente de `b64_json`.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \\
  -H "Authorization: Bearer sk-your-api-key"
```

## Armadilhas Comuns

- Não assuma que todo modelo de imagem é síncrono.
- Não solicite `b64_json` de tarefas de imagem assíncronas; resultados assíncronos são orientados a URLs.
- Não envie `resolution` para modelos de imagem Gemini apenas de proporção.
- Não use `/v1/images/edits` para fluxos de imagem-para-imagem do Nano Banana.
- Defina os tempos limite do cliente HTTP para pelo menos `120s` para solicitações de imagem de alta resolução síncronas.
