## O que é AI Sonar?

AI Sonar é uma plataforma de agregação de APIs de IA que oferece acesso unificado a **centenas de modelos de IA** de fornecedores como OpenAI, Anthropic, Google, DeepSeek, Moonshot, MiniMax e outros.

## Capacidades Suportadas

O catálogo público muda ao longo do tempo. Verifique a disponibilidade atual com `/v1/models`, `recommended_for` ou a página de modelos; os exemplos abaixo usam IDs públicos atuais.

| Capacidade | Descrição | Modelos de exemplo |
|---|---|---|
| Chat e raciocínio | Geração de texto, planejamento, ferramentas | `gpt-5.4`, `claude-sonnet-4-6`, `deepseek-r1` |
| Visão e multimodal | Compreensão de imagens e chat multimodal | `gpt-4o`, `claude-sonnet-4-6`, `gemini-3.5-flash` |
| Geração e edição de imagens | Texto para imagem e edição de imagens | `gpt-image-2`, `flux-pro`, `qwen-image` |
| Geração de vídeo | Texto/imagem/referência para vídeo e controle de movimento | `veo3.1`, `seedance-2.0`, `kling-3.0-video` |
| Áudio e Realtime | TTS, transcrição, tradução e sessões em tempo real | `gpt-4o-mini-tts`, `whisper-1`, `gpt-realtime` |
| Embeddings e rerank | Vetorização de texto e reordenação | `text-embedding-3-small`, `qwen3-vl-rerank` |
| Tradução | Fluxos de tradução de texto e fala | `qwen-mt-plus`, `gpt-realtime-translate` |
| Música e 3D | Geração de música e assets 3D | `suno-music`, `tripo3d-v2.5` |
| Worlds | Geração de worlds, polling de status e media assets | Verifique disponibilidade com `/v1/worlds/generations` e Models API |

## Escolha o formato da requisição

Comece pela rota que corresponde ao comportamento necessário. Use rotas nativas Anthropic/Gemini para recursos nativos do provedor; use /v1/chat/completions compatível com OpenAI ao migrar código de chat no estilo OpenAI.

<CodeGroup>

```bash cURL
curl https://api.aisonar.dev/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.4",
    "messages": [
      {"role": "user", "content": "Explain what AI Sonar does in one sentence."}
    ]
  }'
```

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.aisonar.dev/v1"
)

response = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Explain what AI Sonar does in one sentence."}]
)

print(response.choices[0].message.content)
```

```javascript JavaScript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-your-api-key',
  baseURL: 'https://api.aisonar.dev/v1'
});

const response = await client.chat.completions.create({
  model: 'gpt-5.4',
  messages: [{ role: 'user', content: 'Explain what AI Sonar does in one sentence.' }],
});

console.log(response.choices[0].message.content);
```

<Note>
Use `POST /v1/responses` apenas quando você explicitamente precisar do comportamento específico do Responses. Não é o caminho de compatibilidade padrão para todos os modelos.
</Note>

</CodeGroup>
