---

## Parâmetros de Caminho

<ParamField path="id" type="string" required>
  O ID da tarefa retornado pela solicitação de criação de vídeo. Considere `id` e `task_id` como a mesma identidade assíncrona.
</ParamField>

<Info>
Se uma resposta de criação retornar `poll_url`, chame exatamente essa URL. Quando ela apontar para `/v1/tasks/{id}`, trate isso como o endpoint canônico e fixo de status.

Se a tarefa não existir mais ou não puder mais ser resolvida através do contrato público async-task, AI Sonar retorna `async_task_not_found` com a mensagem `Task not found or no longer available.`
</Info>

## Resposta

<ResponseField name="id" type="string">
  Identificador canônico da tarefa assíncrona.
</ResponseField>

<ResponseField name="task_id" type="string">
  Alias do identificador da tarefa assíncrona.
</ResponseField>

<ResponseField name="poll_url" type="string">
  URL de polling preferencial quando a resposta de criação fornecer uma.
</ResponseField>

<ResponseField name="billing_transaction_id" type="string">
  ID de transação de faturamento da AI Sonar quando a liquidação já foi concluída. Este é o identificador usado no dashboard / conciliação e é separado do `id` / `task_id` assíncrono.
</ResponseField>

<ResponseField name="status" type="string">
  Status da tarefa: `pending`, `processing`, `completed`, `failed`.
</ResponseField>

<ResponseField name="progress" type="number">
  Percentual de progresso (0-100).
</ResponseField>

<ResponseField name="video_url" type="string">
  URL do vídeo gerado (quando concluído).
</ResponseField>

<ResponseField name="video" type="object">
  Objeto único de vídeo com `url`, `duration`, `width` e `height` quando disponíveis.
</ResponseField>

<ResponseField name="videos" type="array">
  Múltiplos objetos de vídeo quando o upstream retornar mais de uma saída.
</ResponseField>

<ResponseField name="error" type="string">
  Mensagem de erro (em caso de falha).
</ResponseField>

<ResponseField name="created" type="integer">
  Timestamp de criação.
</ResponseField>

<ResponseField name="updated" type="integer">
  Timestamp da última atualização.
</ResponseField>

<ResponseField name="model" type="string">
  Modelo usado para a tarefa.
</ResponseField>

```python Python
import requests
import time

task_id = "ldtask_aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa"

while True:
    response = requests.get(
        f"https://api.aisonar.dev/v1/tasks/{task_id}",
        headers={"Authorization": "Bearer sk-your-api-key"}
    )

    data = response.json()
    print(f"Status: {data['status']}, Progress: {data.get('progress', 0)}%")

    if data["status"] == "completed":
        print(f"Video URL: {data['video_url']}")
        break
    elif data["status"] == "failed":
        print(f"Error: {data['error']}")
        break

    time.sleep(5)  # Poll every 5 seconds
```

## Melhores práticas de polling

- Realizar polling a cada 5-10 segundos
- Implemente backoff exponencial para tarefas longas
- Defina um tempo máximo de espera (por exemplo, 10 minutos)
- Trate o status `failed` de forma adequada

```python
import time

def wait_for_video(task_id, max_wait=600, interval=5):
    """Wait for video with timeout."""
    start = time.time()

    while time.time() - start < max_wait:
        response = requests.get(
            f"https://api.aisonar.dev/v1/tasks/{task_id}",
            headers={"Authorization": "Bearer sk-your-api-key"}
        )
        data = response.json()

        if data["status"] == "completed":
            return data["video_url"]
        elif data["status"] == "failed":
            raise Exception(data.get("error", "Video generation failed"))

        time.sleep(interval)

    raise TimeoutError("Video generation timed out")
```
