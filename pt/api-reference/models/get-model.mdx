---
title: "Obter modelo"
openapi: "GET /v1/models/{model}"
description: "Recupera uma instância de modelo"
---

## Parâmetros de rota

<ParamField path="model" type="string" required>
  O ID do modelo a ser recuperado (por exemplo, `gpt-5.4`, `claude-sonnet-4-6`).
</ParamField>

## Resposta

<ResponseField name="id" type="string">
  Identificador do modelo.
</ResponseField>

<ResponseField name="object" type="string">
  Sempre `model`.
</ResponseField>

<ResponseField name="created" type="integer">
  Timestamp de criação.
</ResponseField>

<ResponseField name="owned_by" type="string">
  Provedor do modelo.
</ResponseField>

<ResponseField name="aisonar" type="object">
  Metadados públicos específicos da AI Sonar. A rota de detalhe pode incluir `category`, `pricing_unit`, `capability_flags`, `supported_operations`, `pricing_provenance`, `public_contract_summary`, o `public_contract` completo e `agent_preferences` em nível de cena para instantâneos de recomendação não chat.
</ResponseField>

<RequestExample>
```bash cURL
curl "https://api.aisonar.dev/v1/models/gpt-5.4" \
  -H "Authorization: Bearer sk-your-api-key"
```

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.aisonar.dev/v1"
)

model = client.models.retrieve("gpt-5.4")
print(f"Model: {model.id}, Provider: {model.owned_by}")
```

```javascript JavaScript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-your-api-key',
  baseURL: 'https://api.aisonar.dev/v1'
});

const model = await client.models.retrieve('gpt-5.4');
console.log(`Model: ${model.id}, Provider: ${model.owned_by}`);
```

```go Go
package main

import (
    "encoding/json"
    "fmt"
    "net/http"
)

func main() {
    req, _ := http.NewRequest("GET", "https://api.aisonar.dev/v1/models/gpt-5.4", nil)
    req.Header.Set("Authorization", "Bearer sk-your-api-key")

    resp, _ := http.DefaultClient.Do(req)
    defer resp.Body.Close()

    var model map[string]interface{}
    json.NewDecoder(resp.Body).Decode(&model)
    fmt.Printf("Model: %s, Provider: %s\n", model["id"], model["owned_by"])
}
```

```php PHP
<?php
$ch = curl_init('https://api.aisonar.dev/v1/models/gpt-5.4');

curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_HTTPHEADER => [
        'Authorization: Bearer sk-your-api-key'
    ]

$response = curl_exec($ch);
curl_close($ch);

$model = json_decode($response, true);
echo "Model: {$model['id']}, Provider: {$model['owned_by']}\n";
```

## Tratamento de erros

Se o modelo não existir, você receberá um erro 404:

```json
{
  "error": {
    "message": "Model 'invalid-model' not found",
    "type": "invalid_request_error",
    "code": "model_not_found"
  }
}
```

## Metadados de recomendação para não chat

Para modelos não chat, a rota de detalhe também retorna `aisonar.agent_preferences.<scene>`.

Use isso para inspecionar:

- `preferred_rank`
- `success_rate_24h`
- `sample_count_24h`
- `status`
- `updated_at`

A resposta pública mantém `basis.channel_id` e `basis.physical_model` como `null` de propósito, para que os agentes consumam o sinal de recomendação sem aprender identificadores internos de roteamento.

## Metadados públicos de detalhe

Em comparação com `GET /v1/models`, a rota de detalhe também pode retornar:

- `aisonar.capability_flags`
- `aisonar.supported_operations`
- `aisonar.pricing_provenance`
- `aisonar.public_contract_summary`
- `aisonar.public_contract`
