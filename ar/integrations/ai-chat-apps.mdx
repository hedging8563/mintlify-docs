## نظرة عامة

تتصل معظم تطبيقات دردشة AI العامة بـ AI Sonar من خلال مسار الدردشة القياسي المتوافق مع OpenAI.

عمليًا، هذا يعني:
- استخدم `https://api.aisonar.dev/v1` باعتباره عنوان URL الأساسي ما لم يطلب التطبيق صراحةً عنوان URL كاملًا لنقطة النهاية
- استخدم مفتاح API الخاص بـ AI Sonar كمفتاح API بأسلوب OpenAI
- اختر النماذج التي يمكن للتطبيق استخدامها ضمن سير عمل دردشة قياسي

<Note>
عادةً ما تكون تطبيقات الدردشة العامة **موجّهة نحو chat-completions**، وليست عملاء Codex Responses WebSocket. بالنسبة إلى Codex CLI/App، أو Cursor، أو أدوات البرمجة الأخرى التي تحتاج إلى سلوك خاص بـ Responses، استخدم أدلة التكامل المخصصة بدلًا من ذلك.
</Note>

<Note>
**النوع**: تطبيق دردشة

**المسار الرئيسي**: مسار chat-completions متوافق مع OpenAI

**مستوى الدعم**: مسار مدعوم
</Note>

## التطبيقات المدعومة

| التطبيق | المنصة | ملاحظات |
|-------------|----------|-------|
| Open WebUI | الويب، مستضاف ذاتيًا | عميل مستضاف ذاتيًا شائع يدعم اتصالات متوافقة مع OpenAI |
| LibreChat | الويب، مستضاف ذاتيًا | واجهة دردشة متعددة المزوّدات تدعم نقاط نهاية OpenAI المخصصة |
| LobeChat | الويب، سطح المكتب، مستضاف ذاتيًا | دعم قوي لمزوّدات مخصصة |
| Cherry Studio | Windows، macOS، Linux | مسار جيد لمزوّد متوافق مع OpenAI |
| Chatbox | Windows، macOS، Linux | إعداد بسيط متوافق مع OpenAI |
| TypingMind | الويب، سطح المكتب | يستخدم نمط نقطة نهاية مخصص |
| NextChat | الويب، مستضاف ذاتيًا | عميل خفيف بأسلوب ChatGPT مع دعم تجاوز عنوان base URL |
| ChatWise | Windows، macOS | نمط مزوّد مخصص قياسي |

## قيم الاتصال الافتراضية

| الحقل | القيمة |
|-------|-------|
| عنوان URL الأساسي | `https://api.aisonar.dev/v1` |
| مفتاح API | `sk-your-api-key` |
| نوع API | متوافق مع OpenAI |

إذا طلب التطبيق **عنوان URL كاملًا لنقطة النهاية** بدلًا من عنوان URL الأساسي، فاستخدم:
- Chat Completions: `https://api.aisonar.dev/v1/chat/completions`

## أمثلة على إعدادات التطبيقات

<Tabs>
  <Tab title="LobeChat">
    **LobeChat المستضاف ذاتيًا**

    ```bash
    OPENAI_API_KEY=sk-your-api-key
    OPENAI_PROXY_URL=https://api.aisonar.dev/v1
    ```

    **LobeChat Cloud**
    1. افتح **Settings** → **AI Service Provider**
    2. اختر **OpenAI**
    3. أدخل مفتاح API الخاص بـ AI Sonar
    4. عيّن نقطة النهاية المخصصة إلى `https://api.aisonar.dev/v1`
  </Tab>

  <Tab title="Cherry Studio">
    1. افتح **Settings** → **Model Provider**
    2. أضف مزوّدًا **متوافقًا مع OpenAI**
    3. أدخل:
       - **Name**: AI Sonar
       - **API Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
    4. أضف النماذج التي تريد استخدامها
  </Tab>

  <Tab title="Chatbox">
    1. افتح **Settings** → **AI Model**
    2. اختر **OpenAI API Compatible**
    3. أدخل:
       - **API Host**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
       - **النموذج**: `gpt-5-mini` أو `gpt-4o`
  </Tab>

  <Tab title="TypingMind">
    1. افتح **Settings**
    2. اختر **Custom Endpoint**
    3. أدخل:
       - **Endpoint URL**: `https://api.aisonar.dev/v1/chat/completions`
       - **API Key**: `sk-your-api-key`
    4. قم بتكوين النماذج التي تريد إتاحتها
  </Tab>

  <Tab title="Open WebUI">
    1. افتح **Admin Settings** → **Connections**
    2. أضف اتصالًا **متوافق مع OpenAI**
    3. أدخل:
       - **Base URL**: `https://api.aisonar.dev/v1`
       - **API Key**: `sk-your-api-key`
    4. احفظ ثم اختر النماذج التي تريد إتاحتها
  </Tab>

  <Tab title="LibreChat">
    أضف نقطة نهاية متوافقة مع OpenAI في إعداد المزوّد داخل LibreChat ووجّها إلى:

    ```yaml
    customEndpoints:
      - name: AI Sonar
        apiKey: '${OPENAI_API_KEY}'
        baseURL: 'https://api.aisonar.dev/v1'
        models:
          default: ['gpt-5-mini', 'gpt-5.4', 'claude-sonnet-4-6']
          fetch: false
    ```
  </Tab>
</Tabs>

## الإعدادات الافتراضية الأكثر أمانًا للنماذج

| الفئة | النماذج المقترحة |
|----------|------------------|
| الافتراضي العام | `gpt-5.4`, `gpt-5-mini`, `gpt-4o` |
| Anthropic | `claude-sonnet-4-6`, `claude-opus-4-6` |
| Google | `gemini-2.5-flash`, `gemini-2.5-pro` |
| DeepSeek | `deepseek-r1`, `deepseek-v3-2` |

## نصائح

<AccordionGroup>
  <Accordion title="فضّل Base URL عندما يدعمه التطبيق">
    تكون معظم التطبيقات أسهل في التكوين وأقل عرضة للأخطاء عندما توفّر `https://api.aisonar.dev/v1` باعتباره عنوان URL أساسيًا بدلًا من عنوان URL كامل لنقطة النهاية.
  </Accordion>

  <Accordion title="استخدم النماذج الأصغر أولًا">
    ابدأ بـ `gpt-5-mini` أو `gpt-4o` للدردشة اليومية، ثم انقل مهام الاستدلال الأثقل إلى `gpt-5.4` أو `claude-opus-4-6`.
  </Accordion>

  <Accordion title="أبقِ أدوات البرمجة منفصلة">
    غالبًا ما تحتاج تطبيقات الدردشة العامة ووكلاء البرمجة إلى سلوك مختلف لنقاط النهاية. أبقِ أدوات Codex/Cursor-style ضمن صفحات التكامل المخصصة لها.
  </Accordion>
</AccordionGroup>

## استكشاف الأخطاء وإصلاحها

<AccordionGroup>
  <Accordion title="خطأ في الاتصال">
    - تحقّق من أن عنوان URL الأساسي هو بالضبط `https://api.aisonar.dev/v1`
    - أزل الشرطات المائلة اللاحقة إذا كان التطبيق يضيف المسارات تلقائيًا
    - إذا كان التطبيق يريد عنوان URL كاملًا لنقطة النهاية، فاستخدم `https://api.aisonar.dev/v1/chat/completions`
  </Accordion>

  <Accordion title="401 Unauthorized">
    - تحقّق مرة أخرى من مفتاح API الخاص بك
    - أكّد أن المفتاح نشط في لوحة تحكم AI Sonar
    - تأكد من عدم وجود مسافات إضافية أو فواصل أسطر
  </Accordion>

  <Accordion title="النموذج غير متاح">
    - تحقّق من اسم النموذج بدقة
    - جرّب افتراضيًا مدعومًا على نطاق واسع مثل `gpt-5-mini`, `gpt-4o`, أو `claude-sonnet-4-6`
    - تحقّق من التوفر الحالي في وثائق AI Sonar أو لوحة التحكم
  </Accordion>
</AccordionGroup>
