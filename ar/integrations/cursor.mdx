## نظرة عامة



<Note>
**النوع**: أداة برمجة

**المسار الرئيسي**: متوافق مع OpenAI

**مستوى الدعم**: مدعوم مع حدود
</Note>

يدعم [Cursor](https://cursor.sh) مزوّدي bring-your-own-key (BYOK)، لكن مفاتيح API المخصصة تعمل فقط مع نماذج الدردشة القياسية. استخدم AI Sonar هنا كنقطة نهاية متوافقة مع OpenAI لسير عمل المحرر بأسلوب الدردشة، وليس كبديل كامل لكل ميزات نماذج Cursor.

## الإعداد

1. افتح إعدادات Cursor (`Cmd/Ctrl + ,`)
2. انتقل إلى **Models** في الشريط الجانبي الأيسر
3. مرّر لأسفل إلى قسم **OpenAI API Key**
4. أدخل مفتاح API الخاص بـ AI Sonar
5. فعّل **Override OpenAI Base URL**
6. اضبط Base URL على:

```
https://api.aisonar.dev/v1
```

<Warning>
تنطبق مفاتيح API المخصصة في Cursor فقط على نماذج الدردشة القياسية. سيستمر استخدام Tab Completion وغيرها من الميزات المتخصصة التي يديرها Cursor عبر المزوّدين المدمجين في Cursor.
</Warning>

<Warning>
تنطبق آلية مفتاح API المخصص في Cursor فقط على نماذج الدردشة القياسية. الميزات التي تتطلب نماذج متخصصة يديرها Cursor، مثل Tab Completion وبعض التدفقات الخاصة بالاستدلال، ستستمر في استخدام المزوّدين المدمجين في Cursor. كما يشير Cursor إلى أن الطلبات لا تزال تمر عبر الواجهة الخلفية الخاصة به من أجل التجميع النهائي للموجّه.
</Warning>

## اختيار النموذج

بعد الإعداد، يُفضّل استخدام نماذج الدردشة القياسية المتاحة عبر AI Sonar:

### النماذج الموصى بها للبرمجة

| النموذج | الأفضل لـ |
|-------|----------|
| `gpt-5.4` | البرمجة العامة، تصحيح الأخطاء، الدردشة |
| `gpt-5-mini` | تعديلات سريعة وموجّهات خفيفة |

## ما الذي يعمل بشكل أفضل

### تدفقات الدردشة والمحرر

استخدم AI Sonar عبر Cursor من أجل:

- شرح الأكواد
- إصلاح الأخطاء
- اقتراحات إعادة الهيكلة
- إنشاء التوثيق

### ما الذي يبقى على نماذج يديرها Cursor

هذه الميزات لا تنتقل بشكل موثوق إلى نقطة نهاية AI Sonar الخاصة بك:

- إكمال التبويب
- نماذج الدردشة غير القياسية المتخصصة
- الميزات الخاصة بموفّر معين التي لا يفعّلها Cursor إلا على النماذج المدمجة
