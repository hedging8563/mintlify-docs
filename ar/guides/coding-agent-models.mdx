## كيف يعمل توجيه AI Sonar

AI Sonar توجّه طلباتك إلى أفضل مزود متاح تلقائيًا.

### استراتيجيات التوجيه

| الاستراتيجية | السلوك | الأفضل لـ |
|-------------|--------|-----------|
| **PRIORITY** | التوجيه إلى المزود الأعلى أولوية المتاح | التركيز على الموثوقية |
| **COST** | التوجيه إلى أرخص مزود متاح | التركيز على التكلفة |

يمكنك تعيين استراتيجيتك المفضلة في لوحة التحكم.

### الانتقال التلقائي

إذا كان مزود غير متاح أو أعاد خطأ، تُعيد AI Sonar المحاولة تلقائيًا مع المزود التالي:

```
طلب: claude-sonnet-4-6
  → المزود A (أساسي): خطأ 503
  → المزود B (احتياطي): ✓ نجاح
```

يحدث هذا بشفافية — Coding Agent الخاص بك يرى استجابة طبيعية.

### توجيه التنسيق الأصلي

عندما يدعم النموذج المحدد والقناة الموجَّهة ذلك، يمكن لـ AI Sonar توجيه الطلبات عبر التنسيق الأصلي لواجهة API الخاصة بعائلة النموذج لتحقيق توافق أفضل:

| عائلة النموذج | تنسيق API | عنوان URL الأساسي |
|-------------|------------|----------|
| Claude | Anthropic Messages | `https://api.aisonar.dev` |
| GPT | استجابات OpenAI | `https://api.aisonar.dev/v1` |
| Gemini | واجهة Gemini الأصلية للـ API | `https://api.aisonar.dev` أو `https://api.aisonar.dev/v1beta` بحسب مسار العميل |
| DeepSeek | دردشة OpenAI | `https://api.aisonar.dev/v1` |

يمثل Claude Code مسارًا قويًا من نوع Anthropic-native. توجد أيضًا واجهات Gemini-native، لكن Gemini CLI نفسه ما يزال مسار توافق best-effort ولا ينبغي اعتباره سير عمل مستقرًا على المدى الطويل.
