تغطي واجهات برمجة التطبيقات الخاصة بصور AI Sonar عدة عائلات من النماذج بأشكال طلبات مختلفة. استخدم عقد النموذج العام ومرجع واجهة برمجة التطبيقات المحدد لكل نقطة نهاية قبل افتراض أن حقلًا ما يعمل عبر كل نموذج صورة.

## اختر نقطة النهاية

| الهدف | نقطة النهاية | ملاحظات |
|------|----------|-------|
| تحويل النص إلى صورة | `POST /v1/images/generations` | استخدم `prompt` و`model` صريح |
| تحويل الصورة إلى صورة / صورة مرجعية | `POST /v1/images/generations` | استخدم `operation: "image-to-image"` بالإضافة إلى `image_url` أو `image_urls` أو `reference_image_urls` عندما يدعم النموذج ذلك |
| تحرير صورة GPT | `POST /v1/images/edits` | يدعم `image` متعدد الأجزاء، و`image_url` / `image_urls` بتنسيق JSON، والمراجع الرسمية لـ `images[]` للنماذج المدعومة للتحرير |
| تنويعة قديمة | `POST /v1/images/variations` | يفضل استخدام طرق الجيل أو التحرير الحالية للعمل الجديد |
| حالة غير متزامنة | `GET /v1/tasks/{id}` | استخدم `poll_url` أولاً عند وجوده |

## اكتشاف النموذج

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=image" \\
  -H "Authorization: Bearer sk-your-api-key"
```

تختلف معلمات نموذج الصورة كثيرًا. اقرأ عقد النموذج المحدد قبل إرسال `size` أو `quality` أو `aspect_ratio` أو `resolution` أو `async` أو `background` أو `output_format`.

## المدخلات والحقول الخاصة بالنموذج

- يستخدم `gpt-image-2` أسلوب OpenAI لـ `size` و`quality`؛ احذف `size` أو `quality` لترك AI Sonar يستخدم `auto` حيثما كان مدعومًا.
- تستخدم عائلات صور Gemini وNano Banana عمومًا `aspect_ratio`؛ أرسل `resolution` فقط عندما يكشف النموذج المحدد عنه.
- تنتمي طلبات صورة Nano Banana المرجعية إلى `/v1/images/generations` مع `operation: "image-to-image"` و`image_urls`، وليس إلى `/v1/images/edits`.
- لا يقبل `/v1/images/generations` `images[]` أو `file_id` على المستوى الأعلى؛ تلك هي أشكال تدفق التحرير.
- يجب أن تكون عناوين URL للصور البعيدة عناوين URL عامة من نوع `http`/`https`. تجنب الشبكات الخاصة، والاعتمادات المدمجة، والقطع، وعناوين URL الموقعة قصيرة العمر.

## إنشاء الطلب

```bash
curl https://api.aisonar.dev/v1/images/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-image-2",
    "prompt": "صورة منتج نظيفة لفنجان قهوة خزفي على مكتب من الجوز",
    "size": "1024x1024",
    "response_format": "url"
  }'
```

## استعلام النتائج

تُرجع بعض طرق الصور `data[].url` أو `data[].b64_json` في السطر. تُرجع طرق أخرى `status: "pending"` و`task_id` و`poll_url`. إذا طلبت توليد صورة غير متزامنة أو تحريرها، فإن نتائج المهام المكتملة تُرجع عناوين URL؛ استخدم الطلبات المتزامنة عندما تحتاج تحديدًا إلى `b64_json`.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \\
  -H "Authorization: Bearer sk-your-api-key"
```

## الأخطاء الشائعة

- لا تفترض أن كل نموذج صورة متزامن.
- لا تطلب `b64_json` من مهام الصور غير المتزامنة؛ النتائج غير المتزامنة موجهة نحو URL.
- لا ترسل `resolution` إلى نماذج صور Gemini التي تعتمد فقط على النسبة.
- لا تستخدم `/v1/images/edits` لتدفقات Nano Banana من صورة إلى صورة.
- اضبط مهلات عميل HTTP على الأقل إلى `120s` لطلبات الصور عالية الدقة المتزامنة.
