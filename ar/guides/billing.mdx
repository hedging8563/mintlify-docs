## نظرة عامة

تستخدم AI Sonar **نظام تسعير الدفع حسب الاستخدام**. أنت تدفع فقط مقابل ما تستخدمه، دون اشتراكات أو التزامات دنيا.

## كيف تعمل الفواتير

1. **إضافة رصيد** إلى حسابك
2. **استخدام الـ API** - يتم خصم التكاليف لكل طلب
3. **مراقبة الاستخدام** في لوحة التحكم الخاصة بك
4. **شحن الرصيد** عندما ينخفض رصيدك

## نماذج التسعير

قد تتغير الأسعار المباشرة مع تغيّر المزوّدين والمسارات وعقود النماذج. اعتبر Dashboard وصفحة النماذج و`GET /v1/models/:model/pricing` و[Pricing API](/api-reference/pricing/get-pricing) مصدر الحقيقة.

### التسعير المعتمد على Tokens

تُسعّر معظم نماذج المحادثة والاستدلال وembedding وrerank وبعض نماذج الصور حسب tokens الإدخال أو الإخراج أو الكاش أو إخراج الصورة.

| عائلة التسعير | أمثلة | التحقق من السعر الحالي |
|---|---|---|
| Chat / Responses | `gpt-5.4`, `claude-sonnet-4-6`, `gemini-3.5-flash` | 模型页或 Pricing API |
| 嵌入 / rerank | `text-embedding-3-small`, `qwen3-vl-rerank` | 模型价格详情 |
| 按 token 计费的图像 | `gpt-image-2` | `GET /v1/models/gpt-image-2/pricing` |

<Note>
لا تنسخ جداول أسعار ثابتة إلى منطق الإنتاج. خزّن model IDs فقط في الكود وراجع الأسعار الحالية قبل الإطلاق.
</Note>

### تسعير الطلبات والمهام

قد تُسعّر نماذج الصور والفيديو والموسيقى و3D والصوت وworld حسب الطلب أو الصورة أو الثانية/الدقيقة أو المهمة أو استخدام خاص بالمزوّد.

| العائلة | أمثلة |
|---|---|
| 图像 | `flux-pro`, `qwen-image` |
| 视频 | `veo3.1`, `seedance-2.0` |
| 音乐 | `suno-music` |
| 3D | `tripo3d-v2.5` |
| 音频 | `tts-1`, `whisper-1` |

### فواتير المهام غير المتزامنة Async Task (الفيديو والموسيقى و3D وبعض نماذج الصور)

<Note>
  في تدفقات التوليد القائمة على المهام، قد يؤدي إنشاء المهمة إلى حجز أو خصم مسبق للتكلفة التقديرية. تتم التسوية النهائية فقط بعد أن تصل المهمة غير المتزامنة إلى حالة نجاح نهائية أثناء الاستعلام أو التصفية النهائية.
</Note>

بالنسبة لتدفقات التوليد القائمة على المهام (الفيديو والموسيقى و3D وبعض نماذج الصور):

1. أرسل المهمة. قد تجري AI Sonar حجزاً أو خصماً تقديرياً للتحقق من الرصيد وحدود إنفاق API Key.
2. استعلم عن `poll_url` المرتجع، أو استدعِ `GET /v1/tasks/{id}`، حتى تصل المهمة إلى حالة نهائية.
3. عند اكتمال المهمة بنجاح، تسجل التسوية النهائية الاستخدام وتتضمن استجابة المهمة `billing_transaction_id`.
4. إذا فشل الإنشاء أو كانت الحالة النهائية failed، يتم رد أو تحرير المبلغ المعلّق ويُوسم الطلب بأنه غير قابل للفوترة.

إذا لم يعكس dashboard التسوية أو الاسترداد بعد ظهور الحالة النهائية، فتواصل مع support@aisonar.dev للحصول على المساعدة.

```python
# مثال: فوترة توليد الفيديو
response = client.post("/v1/videos/generations", json={
    "model": "veo3.1",
    "prompt": "A sunset over the ocean"
})
# قد يتم حجز التكلفة التقديرية الآن؛ تظهر الفوترة النهائية بعد النجاح.

poll_url = response.json()["poll_url"]
task_id = response.json()["task_id"]
# استعلم عن poll_url للحالة؛ يظهر billing_transaction_id بعد التسوية.
```

### معرّفات معاملات الفوترة

تتضمن استجابات JSON القابلة للفوترة وغير المتدفقة والمتوافقة مع OpenAI الحقل `billing_transaction_id` عندما تكتمل التسوية قبل إنهاء الاستجابة. وتُعرَض القيمة نفسها أيضًا في ترويسة الاستجابة `X-Billing-Transaction-ID` لتكاملات المتصفح والخادم. قد تعرض مسارات التوافق الأصلية مثل Gemini `/v1beta` القيمة عبر الترويسة فقط للحفاظ على شكل استجابة المزوّد الأصلي. بالنسبة إلى مهام الوسائط غير المتزامنة، استعلم عن `poll_url` المُعاد أو `GET /v1/tasks/{id}`؛ ستتضمن استجابة المهمة `billing_transaction_id` بعد اكتمال التسوية. قد تتم تسوية الاستجابات المتدفقة بعد إرسال التدفق، لذلك إذا لم تكن الترويسة موجودة فاستخدم سجلات الاستخدام في dashboard للمطابقة.

## حساب الـ Tokens

الـ Tokens هي الوحدات الأساسية لمعالجة النصوص:

- ~4 أحرف = 1 token (الإنجليزية)
- ~1-2 أحرف = 1 token (الصينية)
- صورة واحدة = تختلف حسب الحجم والتفاصيل

### تقدير الـ Tokens

```python
# Rough estimation
def estimate_tokens(text):
    return len(text) / 4  # Approximate for English

# Actual count (for OpenAI models)
import tiktoken
encoder = tiktoken.encoding_for_model("gpt-4o")
tokens = encoder.encode("Your text here")
print(f"Token count: {len(tokens)}")
```

## تتبع الاستخدام

### لوحة التحكم

راقب استخدامك في [لوحة التحكم](https://console.aisonar.dev):

- الرصيد في الوقت الفعلي
- سجل الاستخدام حسب النموذج
- تفاصيل التكاليف
- استخدام مفتاح الـ API

### استجابة الـ API

تتضمن كل استجابة معلومات الاستخدام:

```json
{
  "usage": {
    "prompt_tokens": 50,
    "completion_tokens": 100,
    "total_tokens": 150
  }
}
```

## تنبيهات انخفاض الرصيد

تكوين التنبيهات عندما ينخفض رصيدك:

1. انتقل إلى **لوحة التحكم ← الإعدادات ← الإشعارات**
2. حدد مبلغ الحد الأدنى الخاص بك
3. تلقي إشعارات البريد الإلكتروني

## إضافة رصيد

### طرق الدفع

- Stripe (Visa, Mastercard)

### الخطوات

1. تسجيل الدخول إلى [لوحة التحكم](https://console.aisonar.dev)
2. انقر فوق **إضافة رصيد**
3. اختر المبلغ وطريقة الدفع
4. أكمل عملية الدفع

يتم إضافة الرصيد فوراً بعد تأكيد الدفع.

## حدود مفتاح الـ API

يمكنك تعيين حدود للإنفاق على مفاتيح API الفردية:

1. انتقل إلى **لوحة التحكم ← مفاتيح API**
2. انقر فوق مفتاح للتعديل
3. تعيين **حد الاستخدام**

عند الوصول إلى الحد الأقصى، ستُرجع الطلبات باستخدام هذا المفتاح الخطأ `402 Payment Required`.

## الفواتير

بالنسبة لتدفقات حسابات الشركات، تتوفر الفواتير:

1. انتقل إلى **لوحة التحكم ← الفواتير**
2. عرض سجل المعاملات
3. تحميل الفواتير بصيغة PDF
