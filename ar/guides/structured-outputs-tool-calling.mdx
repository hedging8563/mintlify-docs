المخرجات المنظمة والأدوات هي مسارات حقيقية لواجهة برمجة التطبيقات AI Sonar، لكنها ليست قدرة عالمية واحدة عبر كل نموذج ومسار. اختر المسار الذي يمتلك السلوك الذي تحتاجه.

## JSON والمخرجات المنظمة

للحصول على استجابات منظمة قابلة للنقل، ابدأ مع إكمالات الدردشة `response_format: {"type":"json_object"}` وتحقق من المخرجات في تطبيقك. هذا هو الشكل العام الثابت للطلب في التحقق المشترك لـ AI Sonar.

```json
{
  "model": "gpt-5.4",
  "messages": [{"role": "user", "content": "Return a JSON object with city and weather."}],
  "response_format": {"type": "json_object"}
}
```

توجد مسارات المخرجات المنظمة `json_schema` و `strict` في بعض التعليمات البرمجية العلوية والتحويل، لكنها تعتمد على النموذج والمسار. اعتبرها قدرة للتحقق مع النموذج المحدد، وليس كوعود شاملة عبر كل مسار في AI Sonar.

## حدود استدعاء الأدوات

| شكل الأداة | المسار المفضل | ملاحظة الإنتاج |
|------------|-----------------|-----------------|
| أدوات وظائف OpenAI | `/v1/chat/completions` | قابلة للنقل لاستدعاءات الوظائف المعرفة من قبل المستخدم |
| أدوات استجابات OpenAI | `/v1/responses` | استخدمها لأدوات مضمنة أو مستضافة خاصة بالاستجابات |
| أدوات Anthropic | `/v1/messages` | استخدمها للاستخدام الأصلي لأدوات Claude وتدفقات التفكير |
| إعلانات وظائف Gemini / الأدوات المدمجة | `/v1beta/models/:model:generateContent` | احتفظ بحقول الأدوات الأصلية لـ Gemini على مسار Gemini |

لا تنفذ AI Sonar وظيفتك نيابة عنك. عندما يعيد نموذج `tool_calls` أو `function_call` أو `tool_use` أو أجزاء استدعاء وظيفة Gemini، يجب على تطبيقك تشغيل الأداة، وإضافة نتيجة الأداة بالشكل المطلوب من قبل المسار، ومتابعة المحادثة.

## الأخطاء الشائعة

- لا ترسل الأدوات المدمجة الأصلية لمزود الخدمة عبر إكمالات الدردشة وتتوقع أن يتم ترجمتها بصمت.
- لا تعتمد على JSON Schema الصارم ما لم يتم اختبار المسار والنموذج المحددين.
- احتفظ بالمخططات صغيرة وصريحة؛ المخططات الكبيرة تضيف رموزًا وقد تقلل من الموثوقية.
- تحقق من مخرجات النموذج على الخادم حتى عند تمكين وضع JSON.
- اعتبر أخطاء الحقول غير المدعومة كتعليقات على العقد. قم بإزالة الحقل أو استخدم المسار الأصلي الذي يمتلكه.

## مثال

```bash
curl https://api.aisonar.dev/v1/chat/completions \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-5.4",
    "messages": [{"role":"user","content":"Extract name and email as JSON."}],
    "response_format": {"type":"json_object"},
    "tools": [{
      "type": "function",
      "function": {
        "name": "lookup_customer",
        "description": "Look up a customer by email",
        "parameters": {
          "type": "object",
          "properties": {"email": {"type": "string"}},
          "required": ["email"]
        }
      }
    }]
  }'
```
