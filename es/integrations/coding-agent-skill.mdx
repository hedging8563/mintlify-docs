## Qué hace el skill

- Genera ejemplos mínimos ejecutables para chat, imagen, audio, video, traducción y otras familias de API de AI Sonar.
- Usa `https://api.aisonar.dev/v1` para clientes compatibles con OpenAI y explica cuándo cambiar a rutas nativas de Anthropic o Gemini.
- Descubre modelos con `/v1/models`, `/llms.txt` y listas `recommended_for` en lugar de adivinar desde listas obsoletas.
- Lee el contrato público de un modelo antes de reintentar solicitudes no-chat para no eliminar silenciosamente campos no compatibles.
- Gestiona pistas de error Agent-First como `did_you_mean`, `suggestions`, `retry_after` y `recommended_request`.

## Instalación

<Note>
Usa el comando canónico de instalación no interactiva:

```bash
npx skills add https://github.com/hedging8563/aisonar-skills --skill aisonar-api-integration -y
```

Esto instala el skill compartido `aisonar-api-integration` desde el repositorio AI Sonar skills.

Si tu herramienta no soporta el installer, copia `skills/aisonar-api-integration/` del repositorio al directorio compartido de skills o rules de tu herramienta.
</Note>

</Tip>

## Flujo recomendado para Agent

1. Empieza con el ejemplo funcional más pequeño que coincida con el lenguaje y la familia de API solicitados.
2. Si la elección de modelo está abierta, llama a `/v1/models` o `https://api.aisonar.dev/llms.txt` antes de fijar un modelo.
3. Para trabajo no-chat, llama a `/v1/models?recommended_for=<scene>` donde `<scene>` es `image`, `video`, `music`, `3d`, `tts`, `stt`, `embedding`, `rerank` o `translation`.
4. Antes de cambiar una solicitud no-chat fallida, lee `/v1/models/:model` y alinea `supported_operations`, `supported_parameters`, `request_endpoint`, `request_shape_mode` y `recommended_request`.
5. Si la API devuelve un error Agent-First, usa los campos estructurados para corregir la solicitud y reintenta solo si el error es retryable.

## Ejemplo mínimo de chat

Este ejemplo usa el SDK Python de OpenAI con la base URL de AI Sonar:

```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["TOKENLAB_API_KEY"],
    base_url="https://api.aisonar.dev/v1"
)

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```

Ejecútalo con:

```bash
pip install openai
export TOKENLAB_API_KEY="sk-your-api-key"
python app.py
```

## Descubrimiento de modelos y contratos

Prefiere descubrimiento en vivo antes que listas empaquetadas obsoletas:

```bash
# Resumen API legible por máquina
curl https://api.aisonar.dev/llms.txt

# Listar modelos
curl "https://api.aisonar.dev/v1/models" -H "Authorization: Bearer sk-KEY"
curl "https://api.aisonar.dev/v1/models?category=image" -H "Authorization: Bearer sk-KEY"

# Listas cortas clasificadas para no-chat
curl "https://api.aisonar.dev/v1/models?recommended_for=image" -H "Authorization: Bearer sk-KEY"
curl "https://api.aisonar.dev/v1/models?recommended_for=video" -H "Authorization: Bearer sk-KEY"
curl "https://api.aisonar.dev/v1/models?recommended_for=translation" -H "Authorization: Bearer sk-KEY"

# Leer el contrato público de un modelo antes de reintentar solicitudes no-chat
curl "https://api.aisonar.dev/v1/models/gpt-image-2" -H "Authorization: Bearer sk-KEY"

# Detalle solo de precios
curl "https://api.aisonar.dev/v1/models/gpt-image-2/pricing" -H "Authorization: Bearer sk-KEY"
```

## Pistas de enrutamiento nativo

La ruta compatible con OpenAI `/v1` es la opción predeterminada para ejemplos comunes de chat, Responses, imagen, embedding, audio y rerank. Usa rutas nativas de Anthropic o Gemini solo cuando la solicitud necesite comportamiento específico del proveedor o AI Sonar devuelva the native endpoint hint header como pista de optimización.



## Recuperación de errores Agent-First

Los errores incluyen campos que los coding agents pueden analizar sin extraer la documentación:

| Error | Campos útiles | Acción del agent |
|---|---|---|
| Nombre de modelo incorrecto | `did_you_mean`, `suggestions`, `hint` | Reintenta con el modelo corregido o una alternativa sugerida |
| Saldo insuficiente | `balance_usd`, `estimated_cost_usd`, `suggestions` | Pide aprobación o cambia a un modelo asequible |
| Límite de tasa o indisponibilidad temporal | `retryable`, `retry_after`, `alternatives` | Espera, reintenta o elige una alternativa listada |
| Contrato no-chat incompatible | `supported_operations`, `supported_parameters`, `request_endpoint`, `request_shape_mode`, `recommended_request` | Reconstruye la solicitud desde el contrato público y fail closed en campos que cambien la intención |

## Familias de API compatibles

| Familia | Ruta principal |
|---|---|
| Chat y Responses | `/v1/chat/completions`, `/v1/responses` |
| Mensajes nativos Claude | `/v1/messages` |
| Solicitudes nativas Gemini | `/v1beta/models/{model}:generateContent` |
| Imágenes | `/v1/images/generations`, `/v1/images/edits` |
| Video | `/v1/videos/generations` |
| Música | `/v1/music/generations` |
| Worlds | `/v1/worlds/generations` más endpoints de estado world y media asset |
| 3D | `/v1/3d/generations` |
| Audio | `/v1/audio/speech`, `/v1/audio/transcriptions`, `/v1/audio/translations` |
| Realtime | `/v1/realtime?model={model}` |
| Embeddings y rerank | `/v1/embeddings`, `/v1/rerank` |
| Traducción de texto | `/v1/translations` |
