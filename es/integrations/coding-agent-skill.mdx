<Note>
Este skill le enseña a tu agente de programación cómo integrar cientos de **APIs de IA** de AI Sonar simplemente describiendo lo que necesitas. Está empaquetado como un único skill compartido agent-first y funciona con cualquier agente de programación que admita archivos de skills o reglas.

La API es **[Agent-First](/guides/agent-first-api)** — incluso si el agente adivina incorrectamente el nombre de un modelo, la respuesta de error le indica exactamente cómo corregirlo.
</Note>

<Note>
Esta página documenta una capa reutilizable de empaquetado de skill, no una superficie de compatibilidad de protocolo. Si necesitas configuración de punto de conexión, SDK o cliente, usa las páginas de integración dedicadas para esas herramientas.
</Note>

## Cómo Funciona

1. **Describes lo que necesitas** — "Quiero usar GPT-4 en mi proyecto de Python"
2. **El agente llama a la API** — puede adivinar un nombre de modelo o punto de conexión
3. **Si algo sale mal** — la API devuelve pistas estructuradas (`did_you_mean`, `suggestions`, `retry_after`)
4. **El agente se autocorrige** — lee las pistas y vuelve a intentarlo con los parámetros correctos
5. **Obtienes código funcional** — normalmente en el primer o segundo intento

## Instalación

<Note>
Usa el comando de instalación canónico no interactivo:

```bash
npx skills add https://github.com/hedging8563/aisonar-skills --skill aisonar-api-integration -y
```

Esto instala el skill compartido `aisonar-api-integration` desde el repositorio de skills de AI Sonar.

Si tu herramienta no admite el instalador, copia la carpeta `skills/aisonar-api-integration/` del repositorio en el directorio compartido de skills o reglas de tu herramienta.
</Note>

### Verificar la Instalación

Pregúntale a tu agente de programación:
```
¿Qué skills están disponibles?
```

Si ves `aisonar-api-integration`, la instalación fue exitosa.

## Obtén tu API Key

<Steps>
  <Step title="Visita AI Sonar">
    Ve a [aisonar.dev](https://aisonar.dev)
  </Step>
  <Step title="Inicia sesión">
    Crea una cuenta o inicia sesión
  </Step>
  <Step title="Obtén la API Key">
    Ve a [Dashboard → API Keys](https://console.aisonar.dev/api) y crea una nueva key
  </Step>
  <Step title="Copia la Key">
    Tu key comienza con `sk-...` — guárdala de forma segura
  </Step>
</Steps>

<Tip>
No necesitas pegar una API key en prompts ni archivos fuente. El skill debe preguntar qué variable de entorno usar y generar código que lea la key desde esa variable.
</Tip>

## Ejemplos

### Chat Completion

**Tú dices:**
```
Quiero usar GPT-4 en mi proyecto de Python
```

**El agente genera:**
```python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["TOKENLAB_API_KEY"],
    base_url="https://api.aisonar.dev/v1"
)

response = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```

Ejecútalo con:

```bash
export TOKENLAB_API_KEY="sk-your-api-key"
python app.py
```

Si el agente adivina un nombre de modelo incorrecto (por ejemplo, `gpt5`), la API devuelve:
```json
{
  "error": {
    "code": "model_not_found",
    "did_you_mean": "gpt-5.4",
    "suggestions": [{"id": "gpt-5.4"}, {"id": "gpt-5-mini"}],
    "hint": "Did you mean 'gpt-5.4'? Use GET https://api.aisonar.dev/v1/models to list all available models."
  }
}
```

El agente lee `did_you_mean`, corrige el nombre del modelo y vuelve a intentarlo — todo automáticamente.

Generate images with Flux in Node.js

## Recuperación de Errores Agent-First

Cada respuesta de error incluye pistas estructuradas que cualquier agente de programación puede analizar. Consulta la [referencia completa de la API Agent-First](/guides/agent-first-api) para obtener más detalles.

| Error | Lo que devuelve la API | Lo que hace el agente |
|-------|---------------------|---------------------|
| Nombre de modelo incorrecto | `did_you_mean` + `suggestions` | Corrige automáticamente y reintenta |
| Saldo insuficiente | `balance_usd` + `suggestions` más económicas | Cambia a un modelo asequible |
| Modelo no disponible | `alternatives` + `retry_after` | Cambia a un modelo disponible |
| Límite de tasa alcanzado | `retry_after` (segundos exactos) | Espera y reintenta |
| Contexto demasiado largo | `suggestions` con modelos más grandes | Cambia a un modelo con mayor contexto |
| Incompatibilidad de contrato en solicitud no chat | `supported_operations`, `supported_parameters`, `request_endpoint`, `recommended_request` | Reconstruye la solicitud desde el contrato público del modelo |

### Optimización de Endpoint Nativo

Cuando llamas a `/v1/chat/completions` con un modelo Claude o Gemini, la API devuelve headers de optimización:



Trata estos headers como sugerencias de ruta, no como un cambio automático obligatorio. Las llamadas nativas de Claude usan `/v1/messages`; las llamadas nativas de Gemini deben seguir el endpoint anunciado por the native endpoint hint header, normalmente `/v1beta/gemini`.

### Descubrimiento de Modelos

No necesitas buscar en la documentación:

```bash
# Resumen de API legible por máquina
curl https://api.aisonar.dev/llms.txt

# List models by category
curl "https://api.aisonar.dev/v1/models?category=chat" -H "Authorization: Bearer sk-KEY"
curl "https://api.aisonar.dev/v1/models?category=image" -H "Authorization: Bearer sk-KEY"

# Lista recomendada para escenarios no chat
curl "https://api.aisonar.dev/v1/models?recommended_for=image" -H "Authorization: Bearer sk-KEY"
curl "https://api.aisonar.dev/v1/models?recommended_for=translation" -H "Authorization: Bearer sk-KEY"

# Lee el contrato público de un modelo antes de reintentar una solicitud no chat
curl "https://api.aisonar.dev/v1/models/gpt-image-2" -H "Authorization: Bearer sk-KEY"

# Filtrar por capacidad
curl "https://api.aisonar.dev/v1/models?tag=coding" -H "Authorization: Bearer sk-KEY"
```

## FAQ

<AccordionGroup>
  <Accordion title="¿El skill no se activa automáticamente?">
    Intenta mencionar "AI Sonar" o "AI Sonar API" en tu solicitud:
    ```
    Quiero integrar GPT-4 en mi proyecto con AI Sonar
    ```
  </Accordion>

  <Accordion title="¿Cómo ayuda la API al agente a encontrar el modelo correcto?">
    La [API Agent-First](/guides/agent-first-api) de AI Sonar devuelve pistas de error estructuradas. Si el agente adivina un nombre de modelo incorrecto, el error incluye `did_you_mean` con el nombre correcto y `suggestions` con alternativas. El agente las lee y se autocorrige en el siguiente intento.
  </Accordion>

  <Accordion title="¿Qué agentes de programación son compatibles?">
    Cualquier agente de programación que admita directorios compartidos de skills o reglas. Cuando el instalador es compatible, `npx skills add` coloca automáticamente el skill compartido `aisonar-api-integration` en la ubicación detectada.
  </Accordion>

  <Accordion title="¿Cómo actualizar el skill?">
    Vuelve a ejecutar el instalador:
    ```bash
    npx skills add https://github.com/hedging8563/aisonar-skills --skill aisonar-api-integration -y
    ```
  </Accordion>
</AccordionGroup>
