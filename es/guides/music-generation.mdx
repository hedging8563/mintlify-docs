## Flujo recomendado

1. Llama a [Listar modelos](/es/api-reference/models/list-models) con `recommended_for=music`.
2. Usa [Crear música](/es/api-reference/music/create-music) para crear una tarea con `prompt` y campos opcionales como `title`, `tags`, `action`, `mv` o ajustes de continuación.
3. Guarda el `id` / `task_id` y el `poll_url` devueltos.
4. Consulta hasta que la tarea sea `completed` o `failed`, y luego lee `audio_url`, `video_url` opcional, `title` y `lyrics`.

## Descubrir modelos

La generación musical es asíncrona. No dependas del nombre del proveedor upstream ni de su ID de tarea; AI Sonar devuelve una identidad de tarea normalizada y una URL de polling.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=music" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Crear una tarea de música

`action: "MUSIC"` crea canciones o instrumentales según el prompt y el modelo. `action: "LYRICS"` es para flujos solo de letras cuando el modelo lo soporta. Usa `continue_clip_id` y `continue_at` solo para continuaciones documentadas por el contrato del modelo.

```bash
curl https://api.aisonar.dev/v1/music/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "suno_music",
    "prompt": "An upbeat synth-pop track with warm vocals and a clean chorus",
    "title": "Morning Static",
    "tags": "synth-pop, upbeat",
    "action": "MUSIC"
  }'
```

## Sondear resultados

Prefiere siempre el `poll_url` devuelto. El fallback estable es [Estado de tarea](/es/api-reference/tasks/get-task-status). Las tareas de música completadas pueden incluir `audio_url`, `video_url`, `title`, `lyrics` y metadatos normalizados del modelo.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
