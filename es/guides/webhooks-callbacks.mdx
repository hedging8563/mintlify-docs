## Descripción general

Entiende los límites de callbacks de AI Sonar, el fallback por sondeo y el diseño para futuros webhooks.

Usa las API públicas de AI Sonar y contratos de modelo; no dependas de detalles de ejecución del proveedor.

## Reglas clave

- Persiste los `task_id`, `poll_url`, callback payload id antes de devolver éxito al usuario.
- Usa las API públicas de AI Sonar y contratos de modelo; no dependas de detalles de ejecución del proveedor.
- Haz que los reintentos sean idempotentes y evita generación o cobro duplicado.

## Checklist de producción

- Persiste los `task_id`, `poll_url`, callback payload id antes de devolver éxito al usuario.
- Usa las API públicas de AI Sonar y contratos de modelo; no dependas de detalles de ejecución del proveedor.
- Haz que los reintentos sean idempotentes y evita generación o cobro duplicado.
- Los logs deben ayudar al soporte sin guardar secretos ni payloads privados.
- Valida el comportamiento en staging antes de cambiar tráfico de producción.
