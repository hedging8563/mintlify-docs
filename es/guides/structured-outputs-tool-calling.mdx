Las salidas estructuradas y las herramientas son rutas reales de la API de AI Sonar, pero no son una capacidad universal en todos los modelos y rutas. Elige la ruta que posea el comportamiento que necesitas.

## JSON Y Salida Estructurada

Para respuestas estructuradas portátiles, comienza con Chat Completions `response_format: {"type":"json_object"}` y valida la salida en tu aplicación. Esta es la forma de solicitud pública estable en la validación compartida de AI Sonar.

```json
{
  "model": "gpt-5.4",
  "messages": [{"role": "user", "content": "Devuelve un objeto JSON con ciudad y clima."}],
  "response_format": {"type": "json_object"}
}
```

Las rutas de salida estructurada `json_schema` y `strict` existen en algún código de upstream y conversión, pero son dependientes del modelo y la ruta. Trátalas como una capacidad para verificar con el modelo seleccionado, no como una promesa general en todas las rutas de AI Sonar.

## Límite de Llamadas a Herramientas

| Forma de herramienta | Ruta a preferir | Nota de producción |
|---------------------|------------------|--------------------|
| Herramientas de función de OpenAI | `/v1/chat/completions` | Portátil para llamadas a funciones definidas por el usuario |
| Herramientas de Respuestas de OpenAI | `/v1/responses` | Utiliza para herramientas integradas o alojadas específicas de Respuestas |
| Herramientas de Anthropic | `/v1/messages` | Utiliza para el uso de herramientas nativas de Claude y flujos de pensamiento |
| Declaraciones de funciones de Gemini / herramientas integradas | `/v1beta/models/:model:generateContent` | Mantén los campos de herramientas nativas de Gemini en la ruta de Gemini |

AI Sonar no ejecuta tu función por ti. Cuando un modelo devuelve `tool_calls`, `function_call`, `tool_use`, o partes de llamada a funciones de Gemini, tu aplicación debe ejecutar la herramienta, agregar el resultado de la herramienta en el formato requerido por la ruta y continuar la conversación.

## Errores Comunes

- No envíes herramientas integradas nativas del proveedor a través de Chat Completions y esperes que sean traducidas silenciosamente.
- No confíes en el JSON Schema estricto a menos que la ruta y el modelo seleccionados hayan sido probados.
- Mantén los esquemas pequeños y explícitos; los esquemas grandes añaden tokens y pueden reducir la fiabilidad.
- Valida la salida del modelo del lado del servidor incluso cuando el modo JSON esté habilitado.
- Trata los errores de campo no soportado como retroalimentación del contrato. Elimina el campo o utiliza la ruta nativa que lo posee.

## Ejemplo

```bash
curl https://api.aisonar.dev/v1/chat/completions \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-5.4",
    "messages": [{"role":"user","content":"Extraer nombre y correo electrónico como JSON."}],
    "response_format": {"type":"json_object"},
    "tools": [{
      "type": "function",
      "function": {
        "name": "lookup_customer",
        "description": "Buscar un cliente por correo electrónico",
        "parameters": {
          "type": "object",
          "properties": {"email": {"type": "string"}},
          "required": ["email"]
        }
      }
    }]
  }'
```
