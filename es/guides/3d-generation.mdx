## Flujo recomendado

1. Llama a [Listar modelos](/es/api-reference/models/list-models) con `recommended_for=3d`.
2. Usa [Crear 3D](/es/api-reference/3d/create-3d) con un `prompt` de texto; agrega `image` o `image_url` solo para modelos que soporten imagen a 3D.
3. Elige campos opcionales como `format`, `quality`, `style` o `seed` según el contrato del modelo.
4. Consulta `poll_url` hasta que el recurso esté listo y descarga `model_url` o URLs específicas como `glb_url` / `fbx_url`.

## Descubrir modelos

La generación 3D está basada en tareas. Usa el catálogo de modelos como fuente de verdad en lugar de asumir un proveedor o modelo fijo.

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \
  -H "Authorization: Bearer sk-your-api-key"
```

## Crear una tarea 3D

Las solicitudes texto a 3D requieren `prompt`. Las solicitudes imagen a 3D agregan `image` en base64 o `image_url`. Usa `format` y `quality` solo cuando el contrato del modelo los exponga.

```bash
curl https://api.aisonar.dev/v1/3d/generations \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "A stylized low-poly robot mascot with clean topology",
    "format": "glb",
    "quality": "standard"
  }'
```

## Sondear resultados

Usa primero el `poll_url` devuelto. El fallback estable es [Estado de tarea](/es/api-reference/tasks/get-task-status). Las tareas 3D completadas devuelven `model_url` y pueden incluir `glb_url` o `fbx_url`.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \
  -H "Authorization: Bearer sk-your-api-key"
```
