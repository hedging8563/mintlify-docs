---
title: "Generación 3D"
description: "Crea activos 3D a partir de texto o imágenes y descarga archivos de modelo completados de los resultados de tareas asíncronas."
---

La generación 3D es asíncrona. `POST /v1/3d/generations` crea una tarea; las respuestas de estado completadas devuelven URLs de activos de modelo descargables.

## Descubrimiento de Modelos

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=3d" \\
  -H "Authorization: Bearer sk-your-api-key"
```

No asumas que cada modelo 3D soporta tanto texto-a-3D como imagen-a-3D. Verifica el contrato del modelo seleccionado antes de enviar `image`, `image_url`, `format`, `quality`, `style` o `seed`.

## Crear Solicitud

```bash
curl https://api.aisonar.dev/v1/3d/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "tripo3d-v2.5",
    "prompt": "Un robot mascota estilizado de baja poligonización con topología limpia",
    "format": "glb",
    "quality": "standard"
  }'
```

Texto-a-3D requiere un `prompt`. Imagen-a-3D utiliza `image` o `image_url` cuando el modelo lo soporta. Prefiere URLs públicas para entradas de producción a menos que la imagen fuente sea privada y tu cliente suba intencionalmente datos en base64.

## Opciones de Salida

- `format` puede solicitar empaquetado de salida como `glb`, `fbx`, `obj` o `usdz` cuando se soporta.
- Los valores de `quality` como `draft`, `standard` o `high` afectan la latencia y el costo cuando el modelo los expone.
- `style` y `seed` son dependientes del modelo; omítelos a menos que el contrato seleccionado los documente.

## Consultar Resultados

Utiliza primero el `poll_url` devuelto. Si necesitas un endpoint fijo, utiliza `GET /v1/tasks/{id}` con el mismo `id` / `task_id` de la respuesta de creación.

Las tareas completadas devuelven `model_url` y pueden incluir URLs específicas de formato como `glb_url`, `fbx_url`, `obj_url` o `usdz_url`. Descarga y almacena el activo en tu propio producto si los usuarios necesitan acceso repetido.

## Errores Comunes

- No esperes un activo 3D en la respuesta de creación; consulta hasta obtener un estado terminal.
- No prometas cada formato para cada modelo.
- No reintentes solicitudes de creación después de un tiempo de espera del cliente sin verificar si la tarea ya existe.
- No expongas los IDs de tareas del proveedor en tu interfaz de usuario.
