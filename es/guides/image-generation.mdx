Las API de imagen de AI Sonar cubren varias familias de modelos con diferentes formas de solicitud. Utiliza el contrato de modelo público y la referencia API específica del endpoint antes de asumir que un campo funciona en todos los modelos de imagen.

## Elegir El Endpoint

| Objetivo | Endpoint | Notas |
|----------|----------|-------|
| Texto a imagen | `POST /v1/images/generations` | Usa `prompt` y un `model` explícito |
| Imagen a imagen / imagen de referencia | `POST /v1/images/generations` | Usa `operation: "image-to-image"` más `image_url`, `image_urls` o `reference_image_urls` cuando el modelo lo soporte |
| Edición de imagen GPT | `POST /v1/images/edits` | Soporta `image` multipart, `image_url` / `image_urls` en JSON, y referencias oficiales `images[]` para modelos de edición soportados |
| Variación heredada | `POST /v1/images/variations` | Prefiere las rutas de generación o edición actuales para nuevo trabajo |
| Estado asíncrono | `GET /v1/tasks/{id}` | Usa `poll_url` primero cuando esté presente |

## Descubrimiento de Modelos

```bash
curl "https://api.aisonar.dev/v1/models?recommended_for=image" \\
  -H "Authorization: Bearer sk-your-api-key"
```

Los parámetros del modelo de imagen difieren mucho. Lee el contrato del modelo seleccionado antes de enviar `size`, `quality`, `aspect_ratio`, `resolution`, `async`, `background` o `output_format`.

## Entradas Y Campos Específicos Del Modelo

- `gpt-image-2` utiliza `size` y `quality` al estilo de OpenAI; omite `size` o `quality` para permitir que AI Sonar use `auto` donde sea soportado.
- Las familias de imágenes Gemini y Nano Banana generalmente utilizan `aspect_ratio`; solo envía `resolution` cuando el modelo seleccionado lo exponga.
- Las solicitudes de imagen de referencia de Nano Banana pertenecen a `/v1/images/generations` con `operation: "image-to-image"` y `image_urls`, no a `/v1/images/edits`.
- `/v1/images/generations` no acepta `images[]` de nivel superior o `file_id`; esos son formatos de flujo de edición.
- Las URL de imágenes remotas deben ser URL públicas `http`/`https`. Evita redes privadas, credenciales incrustadas, fragmentos y URL firmadas de corta duración.

## Crear Solicitud

```bash
curl https://api.aisonar.dev/v1/images/generations \\
  -H "Authorization: Bearer sk-your-api-key" \\
  -H "Content-Type: application/json" \\
  -d '{
    "model": "gpt-image-2",
    "prompt": "Una foto de producto limpia de una taza de café de cerámica sobre un escritorio de nogal",
    "size": "1024x1024",
    "response_format": "url"
  }'
```

## Consultar Resultados

Algunas rutas de imagen devuelven `data[].url` o `data[].b64_json` en línea. Otras rutas devuelven `status: "pending"`, `task_id` y `poll_url`. Si solicitas generación o edición de imagen asíncrona, los resultados de tareas completadas devuelven URL; usa solicitudes sincrónicas cuando necesites específicamente `b64_json`.

```bash
curl "https://api.aisonar.dev/v1/tasks/$TASK_ID" \\
  -H "Authorization: Bearer sk-your-api-key"
```

## Errores Comunes

- No asumas que cada modelo de imagen es sincrónico.
- No solicites `b64_json` de tareas de imagen asíncronas; los resultados asíncronos están orientados a URL.
- No envíes `resolution` a modelos de imagen Gemini solo de relación.
- No uses `/v1/images/edits` para flujos de imagen a imagen de Nano Banana.
- Establece los tiempos de espera del cliente HTTP a al menos `120s` para solicitudes de imagen de alta resolución sincrónicas.
