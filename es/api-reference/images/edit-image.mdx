### Respuesta de tarea asĂ­ncrona

Usa `async: true` con `gpt-image-2` o modelos de ediciĂłn oficiales FLUX/BFL para crear una tarea en lugar de esperar la imagen editada en la solicitud. La respuesta incluye `status: "pending"`, `task_id` y `poll_url`. Consulta `/v1/tasks/{task_id}` hasta que la tarea llegue a `completed` o `failed`.

Las tareas asĂ­ncronas de ediciĂłn solo devuelven las URL finales. Si necesitas datos de imagen `b64_json` sin procesar, usa una solicitud sĂ­ncrona.


Al crear la tarea puede reservarse el importe estimado. Las tareas completadas se cobran por uso real; las fallidas o vencidas liberan o reembolsan la reserva.
