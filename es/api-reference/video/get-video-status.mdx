title: "Obtener estado del video"
openapi: "GET /v1/tasks/{id}"

---

## Parámetros de ruta

<ParamField path="id" type="string" required>
  El ID de la tarea devuelto por la solicitud de creación de video. Trate `id` y `task_id` como la misma identidad asíncrona.
</ParamField>

<Info>
Si una respuesta de creación devuelve `poll_url`, llame exactamente a esa URL. Cuando apunta a `/v1/tasks/{id}`, trátela como el endpoint canónico y fijo de estado.

Si la tarea ya no existe o no puede resolverse a través del contrato público de tareas asíncronas, AI Sonar devuelve `async_task_not_found` con el mensaje `Task not found or no longer available.`
</Info>

## Respuesta

<ResponseField name="id" type="string">
  Identificador canónico de la tarea asíncrona.
</ResponseField>

<ResponseField name="task_id" type="string">
  Alias del identificador de la tarea asíncrona.
</ResponseField>

<ResponseField name="poll_url" type="string">
  URL preferida para sondeo cuando la respuesta de creación la proporciona.
</ResponseField>

<ResponseField name="billing_transaction_id" type="string">
  ID de transacción de facturación de AI Sonar cuando la liquidación ya terminó. Es el identificador de dashboard/contabilidad y es distinto del `id` / `task_id` asíncrono.
</ResponseField>

<ResponseField name="status" type="string">
  Estado de la tarea: `pending`, `processing`, `completed`, `failed`.
</ResponseField>

<ResponseField name="progress" type="number">
  Porcentaje de progreso (0-100).
</ResponseField>

<ResponseField name="video_url" type="string">
  URL del video generado (cuando está completado).
</ResponseField>

<ResponseField name="video" type="object">
  Objeto de video único con `url`, `duration`, `width` y `height` cuando estén disponibles.
</ResponseField>

<ResponseField name="videos" type="array">
  Múltiples cargas útiles de video cuando el upstream devuelve más de una salida.
</ResponseField>

<ResponseField name="error" type="string">
  Mensaje de error (si falla).
</ResponseField>

<ResponseField name="created" type="integer">
  Marca de tiempo de creación.
</ResponseField>

<ResponseField name="updated" type="integer">
  Marca de tiempo de la última actualización.
</ResponseField>

<ResponseField name="model" type="string">
  Modelo usado para la tarea.
</ResponseField>

```python Python
import requests
import time

task_id = "ldtask_aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa"

while True:
    response = requests.get(
        f"https://api.aisonar.dev/v1/tasks/{task_id}",
        headers={"Authorization": "Bearer sk-your-api-key"}
    )

    data = response.json()
    print(f"Status: {data['status']}, Progress: {data.get('progress', 0)}%")

    if data["status"] == "completed":
        print(f"Video URL: {data['video_url']}")
        break
    elif data["status"] == "failed":
        print(f"Error: {data['error']}")
        break

    time.sleep(5)  # Poll every 5 seconds
```

## Mejores prácticas de sondeo

- Consultar cada 5-10 segundos
- Implementar retroceso exponencial para tareas largas
- Establecer un tiempo máximo de espera (p. ej., 10 minutos)
- Manejar el estado `failed` de forma adecuada

```python
import time

def wait_for_video(task_id, max_wait=600, interval=5):
    """Wait for video with timeout."""
    start = time.time()

    while time.time() - start < max_wait:
        response = requests.get(
            f"https://api.aisonar.dev/v1/tasks/{task_id}",
            headers={"Authorization": "Bearer sk-your-api-key"}
        )
        data = response.json()

        if data["status"] == "completed":
            return data["video_url"]
        elif data["status"] == "failed":
            raise Exception(data.get("error", "Video generation failed"))

        time.sleep(interval)

    raise TimeoutError("Video generation timed out")
```
