## ¿Qué es AI Sonar?

AI Sonar es una plataforma de agregación de APIs de IA que te brinda acceso unificado a **cientos de modelos de IA** de proveedores como OpenAI, Anthropic, Google, DeepSeek, Moonshot, MiniMax y más.

  <Card title="Formatos nativos" icon="shuffle">

## Capacidades compatibles

El catálogo público cambia con el tiempo. Verifica la disponibilidad actual con `/v1/models`, `recommended_for` o la página de modelos; los ejemplos de abajo usan IDs públicos actuales.

| Capacidad | Descripción | Modelos de ejemplo |
|---|---|---|
| Chat y razonamiento | Generación de texto, planificación, herramientas | `gpt-5.4`, `claude-sonnet-4-6`, `deepseek-r1` |
| Visión y multimodal | Comprensión de imágenes y chat multimodal | `gpt-4o`, `claude-sonnet-4-6`, `gemini-3.5-flash` |
| Generación y edición de imágenes | Texto a imagen y edición de imágenes | `gpt-image-2`, `flux-pro`, `qwen-image` |
| Generación de video | Texto/imagen/referencia a video y control de movimiento | `veo3.1`, `seedance-2.0`, `kling-3.0-video` |
| Audio y Realtime | TTS, transcripción, traducción y sesiones en tiempo real | `gpt-4o-mini-tts`, `whisper-1`, `gpt-realtime` |
| Embeddings y rerank | Vectorización de texto y reordenamiento | `text-embedding-3-small`, `qwen3-vl-rerank` |
| Traducción | Flujos de traducción de texto y voz | `qwen-mt-plus`, `gpt-realtime-translate` |
| Música y 3D | Generación de música y assets 3D | `suno-music`, `tripo3d-v2.5` |
| Worlds | Generación de worlds, polling de estado y media assets | Verifica disponibilidad con `/v1/worlds/generations` y Models API |

## Elige el formato de solicitud

Empieza por la ruta que coincida con el comportamiento que necesitas. Usa rutas nativas Anthropic/Gemini para funciones nativas del proveedor; usa /v1/chat/completions compatible con OpenAI al migrar código de chat estilo OpenAI.

<CodeGroup>

```bash cURL
curl https://api.aisonar.dev/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-5.4",
    "messages": [
      {"role": "user", "content": "Explain what AI Sonar does in one sentence."}
    ]
  }'
```

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.aisonar.dev/v1"
)

response = client.chat.completions.create(
    model="gpt-5.4",
    messages=[{"role": "user", "content": "Explain what AI Sonar does in one sentence."}]
)

print(response.choices[0].message.content)
```

```javascript JavaScript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-your-api-key',
  baseURL: 'https://api.aisonar.dev/v1'
});

const response = await client.chat.completions.create({
  model: 'gpt-5.4',
  messages: [{ role: 'user', content: 'Explain what AI Sonar does in one sentence.' }],
});

console.log(response.choices[0].message.content);
```

<Note>
Usa `POST /v1/responses` solo cuando necesites explícitamente un comportamiento específico de Responses. No es la ruta de compatibilidad predeterminada para todos los modelos.
</Note>

</CodeGroup>
